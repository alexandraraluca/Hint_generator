2200A - Eating Game

Let $$$m$$$ be the maximum value in $$$a$$$. After $$$n(m-1)$$$ moves, only players with $$$a_i=m$$$ will have dishes remaining, so no other players can win. If $$$a_i=m$$$, then player $$$i$$$ can win if the first player is player $$$(i\bmod n) + 1$$$. Hence, the answer is the number of maximums of $$$a$$$.

Code (C++)
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int n;
    cin >> n;
    vector<int> a(n);
    for (auto &i: a) cin >> i;
    
    int mx = 0, ans = 0;
    for (int i : a) mx = max(i, mx);
    for (int i : a) ans += i == mx;
    cout << ans << '\n';
}

signed main() {
    int t = 1;
    cin >> t;
    while (t--) solve();
}

2200B - Deletion Sort

If $$$a$$$ is non-decreasing at the start, then the answer is trivially $$$n$$$. Otherwise, there exists an $$$i$$$ such that $$$a_i \gt a_{i+1}$$$. We can delete all elements besides these two and then delete $$$a_i$$$ to get to $$$1$$$ remaining element, so the answer is $$$1$$$ in this case.

Code (C++)
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int n;
    cin >> n;
    vector<int> a(n);
    for (auto &i: a) cin >> i;

    if (is_sorted(a.begin(), a.end())) cout << n << '\n';
    else cout << "1\n";
}

signed main() {
    cin.tie(0)->sync_with_stdio(0);
    int t = 1;
    cin >> t;
    while (t--) solve();
}

2200C - Specialty String

Instead of replacing characters with *s, just delete them from the string. Then, the operation becomes deleting two adjacent characters that are equal. AksLolCoding wins if the string becomes empty. Since deleting a pair can never disrupt another pair, the order in which we do the operations does not matter.

We search through the array $$$\frac{n}{2}$$$ times, removing a pair of equal elements whenever we find one. At the end, just output whether the string is empty.

The problem can also be solved in $$$O(n)$$$ using a stack.

Code 1 (C++)
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int n;
    string s;
    cin >> n >> s;
    for (int i = 0; 2*i < n; i++) {
        for (int j = 1; j < size(s); j++) {
            if (s[j] == s[j-1]) {
                s.erase(j-1, 2);
            }
        }
    }
    cout << (s.empty() ? "YES\n" : "NO\n");
}

signed main() {
    int t = 1;
    cin >> t;
    while (t--) solve();
}
Code 2 (C++)
#include <bits/stdc++.h>

        else st.push(c);
    }
    cout << (st.empty() ? "YES\n" : "NO\n");
}

2200D - Portal

Let $$$A$$$ denote the subarray of elements between the two portals and $$$B$$$ denote the remaining elements. The operations can cyclically shift $$$A$$$. They cannot change the order of elements of $$$B$$$, but they can change the position of $$$A$$$ within $$$B$$$. Importantly, an element cannot move from $$$A$$$ to $$$B$$$ or vice versa.

To make the lexicographically minimum permutation, we should cyclically shift $$$A$$$ until its leftmost element is its minimum element. Then, insert $$$A$$$ to the left of the leftmost element in $$$B$$$ that is greater than the minimum of $$$A$$$ (or after $$$A$$$ if no such element exists).

Code (C++)
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int n, x, y;
    cin >> n >> x >> y;
    x--, y--;
    vector<int> a, b;
    for (int i = 0; i < n; i++) {
        int j;
        cin >> j;
        if (i <= x || i > y) a.push_back(j);
        else b.push_back(j);
    }

    // solve
    if (!b.empty()) rotate(b.begin(), min_element(b.begin(), b.end()), b.end());
    int m = b.empty() ? -1 : b[0];
    auto it = a.begin();
    while (it != a.end() && *it < m) it++;
    a.insert(it, b.begin(), b.end());
    for (int i = 0; i < n; i++) {
        cout << a[i] << " \n"[i == n-1];
    }
}

signed main() {
    int t = 1;
    cin >> t;
    while (t--) solve();
}

Hint 1

What elements can the operation not be used on?

Answer

1 and prime numbers.

Hint 2

What first moves ensure that Alice will win?

Answer

Making a small prime number appear in the array after a number with a larger prime factor.

Hint 3

If Alice can win, then she can ensure victory on her first move.

Tutorial

2200E - Divisive Battle

If the $$$a$$$ is non-decreasing, Bob wins immediately. Otherwise:

If there exists an element $$$x$$$ in $$$a$$$ that is divisible by two distinct primes $$$p \lt q$$$, then Alice can ensure her victory by replacing $$$x$$$ with $$$\frac{x}{p}$$$ and $$$p$$$. Since $$$\frac{x}{p}$$$ is a multiple of $$$q$$$, there will always be a multiple of $$$q$$$ that appears before $$$p$$$ and is greater than $$$p$$$, so the array will never be non-decreasing.

If no such element exists, all elements in the array are either $$$1$$$ or prime powers of the form $$$p^e$$$ for some prime $$$p$$$. Define an array $$$b$$$ such that $$$b_i$$$ is $$$1$$$ if $$$a_i=1$$$ and is the unique prime factor of $$$a_i$$$ otherwise. If $$$b_i$$$ is non-decreasing, then Bob will win (since $$$a$$$ will be non-decreasing once no moves are possible). Otherwise, there exists an $$$i$$$ such that $$$b_{i-1} \gt b_i$$$. Alice can replace $$$a_i=p^e$$$ with $$$p$$$ and $$$p^{e-1}$$$ if $$$e \gt 1$$$. By similar logic to the above paragraph, this will make it impossible for Bob to win, so Alice will win.

Code (C++)
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)

int primebase(int x) {
    set<int> s;
    for (int i = 2; i*i <= x; i++) {
        while (x % i == 0) {
            s.insert(i);
            x /= i;
        }
    }
    if (x > 1) s.insert(x);
    if (s.size() > 1) return -1;
    if (s.size() == 0) return 1;
    return *s.begin();
}

void solve() {
    int n;
    cin >> n;
    vector<int> a(n);
    for (auto &i: a) cin >> i;
    
    // solve
    vector<int> b(n);
    for (int i = 0; i < n; i++) b[i] = primebase(a[i]);
    if (is_sorted(all(a))) {
        cout << "Bob\n";
    } else if (*min_element(all(b)) == -1) {
        cout << "Alice\n";
    } else if (is_sorted(all(b))) {
        cout << "Bob\n";
    } else {
        cout << "Alice\n";
    }
}

signed main() {
    int t = 1;
    cin >> t;
    while (t--) solve();
}

Hint 1

Hint 1.1

Answer

She can use all particles with y≥k−1.

Hint 1.2

Of the particles she can use for some k, which should Bessie use?

Answer

The k particles with the largest x values.

Hint 1.3

Process k in decreasing order.

Hint 2

If Bessie has to use a shop particle, which of her own particles should she replace with it?

Answer

The energy-generating particle with smallest x.

Tutorial

2200F - Mooclear Reactor 2

Consider solving the problem without any shop particles. We can iterate on the number of energy generating particles $$$k$$$ from $$$n$$$ to $$$1$$$. We can only use particles with $$$y\geq k-1$$$. Store a set or min heap of the $$$k$$$ largest $$$x$$$ values. As $$$k$$$ decreases, we add particles to the set/heap, removing the smallest element whenever the size of the set/heap exceeds $$$k$$$. We can store the sum of the elements in the set/heap in a variable $$$s$$$. The answer $$$a$$$ will be the max of $$$s$$$ over all values of $$$k$$$.

To handle shop particles, we can run the algorithm again, but using a maximum set size of $$$k-1$$$ instead of $$$k$$$, reserving an extra slot for the shop particle. We can store the sum $$$s$$$ for each $$$k$$$ in an array $$$f$$$. We can then take the prefix max of $$$f$$$ by setting $$$f[k]:=max(f[k],f[k-1])$$$. For each shop particle, we can:

  • choose to not use it, resulting in $$$a$$$ energy, or
  • pair it with any other subset of size at most $$$y+1$$$, resulting in $$$x+f[y+1]$$$ energy.

The answer for each shop particle is just the maximum of these two values.

Code (C++)
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

template<class T> bool smin(T &a, const T b) { return b < a ? a = b, 1 : 0;}
template<class T> bool smax(T &a, const T b) { return a < b ? a = b, 1 : 0;}

void solve() {
    int n, q;
    cin >> n >> q;
    vector<array<int, 2>> a(n);
    for (auto &i : a) cin >> i[1] >> i[0];

    // solve
    sort(a.rbegin(), a.rend());
    int ptr = 0;
    multiset<int> s;
    ll mx = 0, cur = 0;
    vector<ll> pref(n+1);
    for (int k = n; k >= 0; k--) {
        while (ptr < n && a[ptr][0] >= k) {
            cur += a[ptr][1];
            s.insert(a[ptr][1]);
            ptr++;
        }
        while (s.size() > k + 1) {
            cur -= *s.begin();
            s.erase(s.begin());
        }
        smax(mx, cur);
        pref[k] = (s.size() <= k) ? cur : cur - *s.begin();
    }
    for (int k = 1; k <= n; k++) smax(pref[k], pref[k-1]);

    // queries
    while (q--) {
        int x, y;
        cin >> x >> y;
        cout << max(mx, pref[y] + x) << '\n';
    }
}

signed main() {
    int t = 1;
    cin >> t;
    while (t--) solve();
}

2200G - Operation Permutation

Hint 1

Convert each - operation to + and / operation to x.

Hint 2

Fix the permutation. For each +y operation, by how much would the answer change if it was removed?

Answer

The answer would reduce by the product of y and the x operations that come after it in the permutation.

Hint 3

How can you calculate the expected value of the product x operations that come after +y over all permutations?

Tutorial

2200G - Operation Permutation

Replace each - operation with +(-y) and /y with x(1/y) $$$\pmod{10^9+7}$$$.

Consider the contribution of each +y operation to the final answer. Its contribution is equal to the product of $$$y$$$ and the expected value of the product of x operations that come after it. Let there be $$$m$$$ x operations in total. Then, since all permutations are equally likely, there is a $$$\frac{1}{m+1}$$$ chance for there to be $$$k$$$ x operations after the +y operation for all $$$0\leq k\leq m.$$$

For each $$$k$$$, we need to compute the expected value of the sum of a random subset of $$$k$$$ of the $$$m$$$ x operations. To do this, we can use a DP where $$$dp[i][j]$$$ is the sum of the products of all subsets of size $$$j$$$ using only the first $$$i$$$ x operations. If $$$b_i$$$ is the value of the $$$i$$$-th x operation, then the DP transition is $$$dp[i][j]=dp[i-1][j]+b_i\cdot dp[i-1][j-1]$$$. Calculating $$$dp$$$ takes $$$O(m^2)$$$ time.

The expected value of the sum of a random subset of $$$k$$$ x operations is $$$\frac{dp[m][k]}{\binom{m}{k}}$$$, so the expected value of the product of the x operations after a + operation is

$$$$$$S=\frac{1}{m+1}\sum_{k=0}^m \frac{dp[m][k]}{\binom{m}{k}}.$$$$$$

For each +y operation, its contribution to the final answer is $$$S\cdot y$$$. After summing contributions of the +y terms, we just need to add the contribution of $$$x$$$, which is $$$x\cdot dp[m][m]$$$, to get the final answer.

The time complexity is $$$O(n^2)$$$ since $$$m\leq n$$$.

Code (C++)
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

const int mod = 1e9+7;

ll mpow(ll a, ll b) {
    ll r = 1;
    while (b) {
        if (b & 1) {
            r *= a;
            r %= mod;
        }
        b >>= 1;
        a *= a;
        a %= mod;
    }
    return r;
}

ll inv(ll a) {
    return mpow(a, mod-2);
}

const int maxn = 5000;
ll facts[maxn+1] = {1};

void precomp() {
    for (int i = 1; i <= maxn; i++) {
        facts[i] = facts[i-1] * i % mod;
    }
}

void solve() {
    int n;
    ll x;
    cin >> n >> x;

    vector<ll> mults;
    ll adds = 0;
    for (int i = 0; i < n; i++) {
        char c;
        ll v;
        cin >> c >> v;
        if (c == '-') adds += mod - v;
        else if (c == '+') adds += v;
        else if (c == '/') mults.push_back(inv(v));
        else mults.push_back(v);
    }
    adds %= mod;

    // solve
    int m = mults.size();
    vector<ll> dp(m+1);
    dp[0] = 1;
    for (auto &b : mults) {
        x *= b;
        x %= mod;
        for (int i = m; i >= 1; i--) {
            dp[i] += dp[i-1] * b;
            dp[i] %= mod;
        }
    }

    ll S = 0;
    for (int i = 0; i <= m; i++) {
        S += dp[i] * facts[i] % mod * facts[m-i] % mod * inv(facts[m]) % mod;
        S %= mod;
    }
    S *= inv(m+1);
    S %= mod;

    // answer
    ll ans = x + adds * S;
    ans %= mod;
    cout << ans << "\n";
}

signed main() {
    precomp();
    int t = 1;
    cin >> t;
    while (t--) solve();
}

Hint 1

Find a recursive definition for special numbers.

Hint 2

When can xmod42 tell us if x is special?

Answer

Whenever 42∤x.

Hint 3

If 42∣x, how can we determine if x is special?

Answer

x will be special if and only if x42 is special.

Hint 4

Iterate on the number of operations mod42.

Tutorial

2200H - Six Seven

Consider a recursive definition of being special. If a number $$$x$$$ is not divisible by $$$42$$$, it is special if and only if it is divisible by $$$6$$$. Otherwise, since $$$f_6(\frac{x}{42})=f_6(x)-1$$$ and $$$f_7(\frac{x}{42})=f_7(x)-1$$$, $$$x$$$ is special if and only if $$$\frac{x}{42}$$$ is special.

Let the number of operations performed be $$$m$$$. For each value of $$$k=m\bmod{42}$$$, let us find the minimum possible $$$m$$$. For each $$$i$$$, if $$$42\nmid a_i+k$$$, this value of $$$k$$$ is impossible unless $$$6$$$ divides $$$a_i+k$$$. Otherwise, $$$\frac{a_i+m}{42}$$$ must be special. Create an array $$$b$$$ of the values of $$$\frac{a_i+k}{42}$$$ for all $$$i$$$ such that $$$42\mid a_i+k$$$. Finding the minimum $$$m$$$ congruent to $$$k\bmod{42}$$$ is equivalent to solving the original problem for the array $$$b$$$, so we can use recursion to find $$$m'$$$, the minimum number of operations to make all elements in $$$b$$$ special. (Make sure to create a base case in the recursion!) Then, the minimum $$$m$$$ for this case is $$$m=42m'+k$$$. Our final answer is just the minimum of these values over all possible $$$k$$$.

To see why this runs in the time limit, observe that each $$$a_i$$$ corresponds to exactly one array element at each recursion depth. Hence, each recursion depth is processed in a total of $$$O(n)$$$ time. Since each recursion divides values by $$$42$$$ compared to the previous recursion, the maximum recursion depth is $$$O(\log_{42} 10^9)$$$. Thus, the total time complexity is $$$O(n\log_{42} 10^9)$$$, though it may vary based on implementation.

Code (C++)
#include <bits/stdc++.h>
using namespace std;

using ll = long long;
const ll inf = LLONG_MAX / 2;

ll f(const vector<ll>& a) {
    if (a.empty()) return 0;
    if (a.back() == 1) return 5;
    int n = a.size();
    int mod6 = a[0] % 6;
    vector<ll> groups[7];
    for (ll x : a) {
        if (x % 6 != mod6) return inf;
        groups[(x - mod6) / 6 % 7].push_back(x);
    }
    ll ans = inf;
    for (int r = 0; r < 7; r++) {
        ll k = (42 - 6*r - mod6) % 42;
        vector<ll> next;
        for (ll x : groups[r]) next.push_back((x + k) / 42);
        ll rec = f(next);
        if (rec >= inf) continue;
        ans = min(ans, k + 42 * rec);
    }
    return ans;
}

void solve() {
    int n;
    cin >> n;
    vector<ll> a(n);
    for (auto &x : a) cin >> x;
    sort(a.begin(), a.end());
    ll res = f(a);
    if (res >= inf) cout << "-1\n";
    else cout << res << '\n';
}
signed main() {
    cin.tie(0)->sync_with_stdio(0);
    int t = 1;
    cin >> t;
    while (t--) solve();
}

      We are trying to calculate the expected value (EV) of the final value of the expression. To do this, by linearity of expectation, we can calculate the expected value of each additive term separately and sum them all to obtain the final answer.

      At the end of an arbitrary expression, an additive term yy will have the form y⋅Xy⋅X, where XX is the product of all multiplicative terms to the right of yy. There may be many possible values for XX, but only its expected value matters to us.

      Let SS denote E[X]E[X], the expected value of XX. The contribution to the answer for each additive term yy is, therefore, S.yS.y. We now compute SS.

      Again, by linearity of expectation (with a small twist), we have: S=∑mk=0Pr(there are k multiplicative terms to the right of y)⋅E[product if there are k multiplicative terms to the right of y].S=∑mk=0Pr(there are k multiplicative terms to the right of y)⋅E[product if there are k multiplicative terms to the right of y].

      Each probability term is 1m+11m+1 due to randomness, and each expected product term corresponds to the dp/choosedp/choose component.

      That's how the editorial finds S=1m+1∑mk=0dp[m][k]/(mk)S=1m+1∑mk=0dp[m][k]/(mk)

O(n) solution for C

As explained in the editorial, it is always optimal to delete a pair when you find one. You can avoid looping through the string multiple times by mantaining a stack with all currently undeleted characters. Loop through characters in the string from start to end. When you encounter character cc in the string, if it matches the top element of the stack, tt, then cc and tt will eventually become adjacent after a bunch of deletions, so remove tt from the stack. Otherwise, add cc to the stack. If the stack is empty after looping through the string, it's possible to turn the string into stars.

int n;
string s;
cin >> n >> s;
stack<char> st;
for (char c : s) {
	if (!st.empty() && st.top() == c)
		st.pop();
	else
		st.push(c);
}
cout << (st.empty() ? "YES\n" : "NO\n");

A little bit different solution for Task E: -

I used a simulation-based thought process, focusing on the explicit sequences each player tries to construct: -

We know that: -

  • Alice wants to make the array unsorted. Her goal is to create a state where Ai>Aj for some i<j.

  • Bob wants to keep the array sorted. His goal is the exact opposite, ensuring Ai≤Ai+1 for all valid i.

  • Because both players play optimally, for a composite number C with prime factors as p1≤p2≤⋯≤pk is changed in a specific arrangement, by alternately distributing the prime factors into a left and right partition (and reversing the right), the split naturally S is increasing first, then decreasing. Mathematically, this is like a bitonic sequence that looks like this: —

S=(p2,p4,…,ppeak,…,p3,p1)

Combining these observations, we can construct the final modified sequence by replacing every composite number with their respective expanded sequences.

Once we create this fully expanded sequence, our final check is very simple: If the final modified sequence is non-decreasing, Bob wins. Otherwise, Alice wins. There's also a base condition that if the initial sequence is non-decreasing in the first place, then Bob wins.

I hope this helps! :)

Even Palindrome like solution for C

...A lot of cool solutions for C, I just thought of it as an Even Palindrome problem. Used 2 adjacent pointers(i,j) and if they were equal then converted both to '*' & started deacreasing i and increasing j, if string[i/j] is a '*' then just ignore it and decrease/increase i/j respectively till the j > string.size().

int t = 0;
    cin >> t;
    while(t--){
        int n = 0;
        cin >> n;
        string s = "";
        cin >> s;
        // code for even palindrome llmllm
        int i = 0;
        int j = 1;
        while(i >= 0 && j < s.size()){
            while(i >= 0 && j < s.size() && s[i] == s[j]){
                s[i] = '*';
                s[j] = '*';
                i--;
                j++;
                while(i >= 0 && s[i] == '*')
                    i--;
                while(j < s.size() && s[j] == '*')
                    j++;
            }
            if(j > s.size() - 1)
                break;
            i = j;
            j++;
        }
        bool ans = true;
        for(int i = 0; i < s.size(); i++){
            if(s[i] != '*')
                ans = false;
        }
        debug(s);
        ans ? cout << "YES" : cout << "NO";
        cout << endl;
