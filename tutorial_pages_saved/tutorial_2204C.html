2204A - Passing the Ball

Idea: BledDest

Tutorial

2204A - Passing the Ball

It's possible to solve this problem by carefully implementing the process described in the statement: you should code a for-loop to iterate over all iterations, maintain which student has the ball now, and mark all students which have received the ball at least once in some array.

However, there is a much easier (in terms of code) solution. Consider the first character L in the string s; let its index be i. When the student (i−1) passes the ball to the student i, the student i will pasa it back, then the student (i−1) passes the ball back to the student i, and so on, until the ball has been passed n times. So, no student after the i-th one will ever receive the ball, and every student before the i-th one (including him) will receive the ball at least once.

So, the answer depends only on the first occurrence of L in the string.

Solution (BledDest)
#include<bits/stdc++.h>
 
using namespace std;
 
int main()
{
    int t;
    cin >> t;
    for(int i = 0; i < t; i++)
    {
        int n;
        cin >> n;
        string s;
        cin >> s;             
        cout << s.find("RL") + 2 << endl;
    }
    return 0;
}

2204B - Right Maximum

Idea: fcspartakm

Tutorial

2204B - Right Maximum

There are many ways to solve this problem. One approach is to divide the elements into two groups: those that will be selected during the operations (group 1) and those that will not be selected (group 2). The number of operations will then be equal to the size of group 1.

Suppose there is an element strictly greater than some element ai to the left of it. It can be shown that in this case, i will never be selected in the operations: if the element ai still exists, then the element to the left and greater than it also exists, which means that the element ai cannot be the maximum. Thus, every element ai for which there exists an element to the left that is greater belongs to group 2.

On the other hand, if some element ai is greater than or equal to all elements to its left, it cannot be removed without being selected: in each operation, all removed elements, except for the selected one, are strictly less than the selected one, which means that for such an element, there is no way to remove it without selecting it. Therefore, all such elements belong to group 1.

Thus, it is sufficient to check for each element whether there exists a greater element to its left. To do this, you can traverse the array from left to right, keeping track of the maximum among all previously encountered elements. If an element is less than the maximum of all previous ones, it belongs to group 2; otherwise, it belongs to group 1.

Solution (BledDest)
#include<bits/stdc++.h>
 
using namespace std;

void solve()
{
    int n;
    cin >> n;
    vector<int> a(n);
    for(int i = 0; i < n; i++) cin >> a[i];
    int mx = 0;
    int ans = 0;
    for(int i = 0; i < n; i++)
    {
        if(a[i] >= mx) ans++;
        mx = max(mx, a[i]);
    }
    cout << ans << "\n";
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    for(int i = 0; i < t; i++)
        solve();
    return 0;
}

2204C - Spring

Idea: BledDest

Tutorial

2204C - Spring

Let's write a function get(x,y,z,n) that calculates how many liters of water a person will collect over n days if they come every x days, while two other people come every y and z days. The answers to the problem will be get(a,b,c,m), get(b,a,c,m), and get(c,a,b,m).

How do we calculate get(x,y,z,n)? Let's try to divide the days into three categories: those when one person comes, those when two people come, and those when all three come (we will assume that among the people who come, there is always the person who comes every x days).

How do we count the number of days when only the first person comes? At first glance, it seems that there are ⌊nx⌋ of them. However, we will count the days when they come with someone else too; therefore, we need to subtract the number of days when the first and second person come, as well as the number of days when the first and third person come. But we have subtracted the days when all three come too many times, so we need to add that back.

The number of days when two people (including the first) come can be counted similarly: we take all the days when the first and second come, add all the days when the second and third come, and subtract twice the days when all three come (we counted these days twice, but we didn't need to count them at all).

Now we need to understand how to count the number of days when the first and second must come (and possibly the third). These are all days i for which imodx=0 and imody=0; in other words, imodLCM(x,y)=0, where LCM(x,y) is the least common multiple of x and y. Thus, there are exactly ⌊nLCM(x,y)⌋ such days. The number of days when the first and third must come (and possibly the second) is counted similarly.

Finally, we need to count the number of days when all three come. The number of such a day must be divisible by both x, y, and z; that is, it must be divisible by LCM(x,y,z), and there will be ⌊nLCM(x,y,z)⌋ such days.

This is one possible implementation. It can be generalized to a larger number of people using the inclusion-exclusion principle, but that was not required in this problem.

Solution (BledDest)
#include<bits/stdc++.h>
 
using namespace std;
 
long long getOne(long long a, long long m)
{
    return m / a;
}
 
long long getTwo(long long a, long long b, long long m)
{
    return m / lcm(a, b);    
}
 
long long getThree(long long a, long long b, long long c, long long m)
{
    return m / lcm(lcm(a, b), c);
}
 
long long get(long long a, long long b, long long c, long long m)
{
    long long c1 = getOne(a, m);
    long long c2 = getTwo(a, b, m) + getTwo(a, c, m);
    long long c3 = getThree(a, b, c, m);
    return (c1 - c2 + c3) * 6 + (c2 - 2 * c3) * 3 + c3 * 2;
}
 
void solve()
{
    long long a, b, c, m;
    cin >> a >> b >> c >> m;                                                                      
    cout << get(a, b, c, m) << " " << get(b, a, c, m) << " " << get(c, a, b, m) << endl;
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    for(int i = 0; i < t; i++)
        solve();
    return 0;
}

2204D - Alternating Path

Idea: BledDest

Tutorial

2204D - Alternating Path

Note that the direction of edges in one connected component does not affect other components. Therefore, the answer to the problem is the sum of the answers for all connected components.

It is difficult to analyze the beauty of a vertex v by considering infinitely many paths. Let's try to simplify the condition. Let's look at paths of length 2. They indicate that all edges incident to vertex v must be directed away from it towards its neighbors. Now consider paths of length 3: all edges incident to the neighbors of v must be directed towards them. And so on.

This means that the vertices can be divided into two classes based on the parity of the distance from v to them. For even vertices, the incident edges are directed away from them, while for odd vertices, they are directed towards them. Thus, it makes sense that each vertex in a connected component must always have the same parity regardless of the choice of path. If there exist two paths such that the vertex is at an even position in one of them and at an odd position in the other one, then the incident edges must be directed both away from and towards it at the same time, which is impossible.

In other words, a connected component must not contain odd length cycles. This means that the component must be bipartite. If a connected component is not bipartite, then it will always have 0 beautiful vertices.

Let's fix some beatiful vertex v in a bipartite component. Now all edges are uniquely directed. Moreover, it turns out that all vertices in the same part as v are also beautiful. Therefore, in the bipartite component, we can choose a part which size will be added to the answer. Obviously, we should take the larger of the two parts.

Overall complexity: O(n+m).

Solution (awoo)
#include <bits/stdc++.h>
 
#define forn(i, n) for (int i = 0; i < int(n); i++)
 
using namespace std;
 
 
int main() {
	cin.tie(0);
	ios::sync_with_stdio(false);
	int t;
	cin >> t;
	while (t--){
		int n, m;
		cin >> n >> m;
		vector<vector<int>> g(n);
		forn(i, m){
			int v, u;
			cin >> v >> u;
			--v, --u;
			g[v].push_back(u);
			g[u].push_back(v);
		}
		vector<int> clr(n, -1);
		int ans = 0;
		forn(i, n) if (clr[i] == -1){
			queue<int> q;
			q.push(i);
			clr[i] = 0;
			vector<int> cnt(2);
			bool ok = true;
			while (!q.empty()){
				int v = q.front();
				q.pop();
				++cnt[clr[v]];
				for (int u : g[v]){
					if (clr[u] == clr[v])
						ok = false;
					else if (clr[u] == -1){
						clr[u] = clr[v] ^ 1;
						q.push(u);
					}
				}
			}
			if (ok) ans += max(cnt[0], cnt[1]);
		}
		cout << ans << '\n';
	}
	return 0;
}

2204E - Sum of Digits (and Again)

Idea: BledDest

Tutorial

2204E - Sum of Digits (and Again)

We could iterate through the number from which the string was constructed before shuffling, but that would take too long. Instead, let's iterate through the sum of the digits of that number. It will be no greater than 9⋅|s|.

Knowing the sum of the digits of the number, we can construct a fairly short "suffix" of the string without the original number itself. For example, if the sum of the digits of the number is 389, we can understand that the string ends with 389202. If the count of occurrences of each digit in this suffix does not exceed that in the original string (for the original string, we can pre-calculate the count of occurrences of all digits), then we can almost use this suffix. Almost, because we still need to check that the remaining digits sum up to exactly the sum we considered — so that the remaining digits can form a number that gives the desired sum of digits.

If both conditions are met — there are enough digits in the string to form the suffix, and the sum of the remaining digits corresponds to the required one — then the remaining digits can be arranged in non-increasing order (to avoid leading zeros) and prepended to the constructed suffix.

We can check each sum of digits in O(d+m) time, where d is the base of the numeral system (up to 10), and m is the length of the constructed suffix (which does not exceed 11 characters). After that, as soon as we find a suitable sum, we can construct the answer in O(|s|) time.

Thus, we obtain a solution in O(d2⋅|s|+d⋅m⋅|s|).

Solution (BledDest)
#include<bits/stdc++.h>
 
using namespace std;
 
int f(int x)
{
    int res = 0;
    while(x != 0)
    {
        res += x % 10;
        x /= 10;
    }
    return res;
}
 
void solve()
{
    string s;
    cin >> s;
    int n = s.size();
    if(n == 1)
    {
        cout << s << "\n";
        return;
    }
    vector<int> cnt(10);
    int sum_digits = 0;
    for(auto x : s) 
    {
        cnt[x - '0']++;
        sum_digits += x - '0';
    }
    for(int x = 1; x <= n * 9; x++)
    {
        int nx = x;
        string sx = to_string(nx);
        while(nx > 9)
        {
            nx = f(nx);
            sx += to_string(nx);    
        }
        //cout << x << " " << sx << endl;               
        vector<int> need_cnt(10);
        int sum_digits_x = 0;
        for(auto y : sx)
        {
            need_cnt[y - '0']++;
            sum_digits_x += y - '0';
        }
        bool can = true;
        for(int i = 0; i <= 9; i++)
            if(need_cnt[i] > cnt[i]) can = false;
        if(sum_digits - sum_digits_x != x)
            can = false;
        if(can)
        {
            string res = "";
            for(int i = 9; i >= 0; i--)
            {
                for(int j = 0; j < cnt[i] - need_cnt[i]; j++)
                    res.push_back(char('0' + i));
            }
            res += sx;
            cout << res << endl;
            return;
        }
    }
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    for(int i = 0; i < t; i++)
        solve();
    return 0;
}

2204F - Sum of Fractions

Idea: adedalic

Tutorial

2204F - Sum of Fractions

To begin with, let's understand how to obtain the maximum sum MSF(b,k). It can be proven (all proofs are at the end) that it is always optimal to choose the minimum bi and only increase it. Next, it can be shown that if k is sufficient to reduce the denominator to 1, it is advantageous to first reduce the denominator to 1, and then increase the numerator. If k is not sufficient, it is better not to touch the denominator and instead "pour" everything into the numerator.

Thus, we can derive the following formula for MSF. Let bm be the minimum, then

  • if k<bm, then MSF(b,k)=k+1bm+∑i≠m1bi;
  • otherwise (k≥bm) MSF(b,k)=k+(2−bm)+∑i≠m1bi.

Next, since we need to calculate the sum of MSF over all subarrays a, let's understand what contribution each ai makes to the total sum. Suppose that for each subarray a[l…r], we choose its leftmost minimum ai. Then, from the perspective of ai, all elements to the left of it are strictly greater than it, and all elements to the right are greater than or equal to it. That is, for each element ai, we are interested in

  • the rightmost position l′ such that l′<i and al′≤ai;
  • the leftmost position r′ such that r′>i and ar′<ai.
In this case, in any segment a[l…r] such that l′<l≤i and i≤r<r′, the element ai will be the leftmost minimum. In all other segments, it will not be.

As a result, for each ai, there are exactly ci=(i−l′)(r′−i) segments in which it is the minimum, and ri=(i+1)(n−i)−ci (0-indexing) segments where it is not the minimum. If ai is not the minimum, it simply adds ri⋅1ai to all ki.

If ai is the minimum, then

  • to all kj<ai, it adds ci⋅kj+1ai=ciaikj+ciai;
  • to all kj≥ai, it adds ci⋅(kj+2−ai)=cikj+ci(2−ai).
In both cases, we are just adding a linear function to kj, which can be done easily with two prefix sums: it is sufficient to separately store the sum of coefficients in front of k and the sum of the free terms.

Thus, for each ai, we find the first position p such that kp≥ai; for all kj in the interval [0,p), we add ciaikj+ciai; for all kj in the interval [p,m), we add cikj+ci(2−ai).

In the end, the total complexity is O(n(logM+logm)+m), where logM comes from taking the inverse for ai and logm comes from lower_bound.

Proofs:

First, let's define how to increase a fraction most profitably. Suppose we want to increase the fraction 1a by k. In general, we get f(x)=1+k−xa−x, where x is an integer and 0≤x≤min(k,a−1). It is easy to calculate the derivative f′(x)=1+k−a(a−x)2 and understand that for k<a, on the interval [0,k], the value of f′(x)≤0, which means the maximum is at x=0. Conversely, for k≥a, on [0,a−1], the value of f′(x)>0 and the maximum is at x=a−1. Thus, we have proven that for k<a, the maximum fraction is of the form 1+ka, and for k≥a, it is 1+k−a+11=k+2−a.

Secondly, we will prove that it is always beneficial to choose the fraction with the minimum bm and only increase it. To do this, let's consider two cases. Case 1: after a total increase of k, all fractions are less than 1.

By contradiction: suppose that besides the m-th fraction, we also increased some other fraction i: the m-th fraction by x≥0, and the i-th fraction by y>0. Notice that it is always better to increase m by x+1 and i by y−1: 1+x+1bm+1+y−1bi=1+xbm+1+ybi+1bm−1bi≥1+xbm+1+ybi.

Case 2: There is at least one fraction that is not less than 1. First, if the m-th fraction is less than 1 (i.e., there is another fraction i that is not less than 1), it is beneficial to "swap their increases" x and y (thereby increasing the m-th fraction to one), since 1+xbm+(y+2−bi)≥y+2−bm+1+xbi⇔1+xbm−1+xbi≤1+xbm≤1≤bi−bm.

Secondly, when there is one fraction that is not less than 1, it is beneficial to continue increasing that fraction. Specifically, if we take one unit of increase from any other fraction — its value will decrease by no more than 1, while increasing a fraction that has already reached one will increase it strictly by 1, meaning the total sum will not decrease.

Solution (adedalic)

#define sz(a) int((a).size())
 
#define x first

	a += b;
	while (a >= MOD)
		a -= MOD;
	while (a < 0)
		a += MOD;

}

}
int binPow(int a, int k) {
	int ans = 1;
	while (k > 0) {
		if (k & 1)
			ans = mul(ans, a);
		a = mul(a, a);
		k >>= 1;

inline void solve() {
	vector<int> lf(n, -1), rg(n, n);
	vector<int> st;
	fore (i, 0, n) {
		while (!st.empty() && a[st.back()] > a[i])
			st.pop_back();
		if (!st.empty())
			lf[i] = st.back();
		st.push_back(i);
	}
	st.clear();
	for (int i = n - 1; i >= 0; i--) {
		while (!st.empty() && a[st.back()] >= a[i])
			st.pop_back();

			rg[i] = st.back();

		pS[l] = add(pS[l], val);
		pS[r] = add(pS[r], -val);
	};
 
	fore (i, 0, n) {
		int cnt = mul(i - lf[i], rg[i] - i);

	}

		la[i + 1] = add(la[i + 1], la[i]);
		lk[i + 1] = add(lk[i + 1], lk[i]);
		ra[i + 1] = add(ra[i + 1], ra[i]);
		rk[i + 1] = add(rk[i + 1], rk[i]);
	}
 
	fore (i, 0, m) {
		int ans = commonSum;
		ans = add(ans, mul(add(la[i], ra[i]), k[i]));
		ans = add(ans, add(lk[i], rk[i]));
		cout << ans << '\n';
	}
}

2204G - Grid Path

Idea: Roms

Tutorial

2204G - Grid Path

Since we can only move vertically downwards and not upwards, in each visited row, the traversed cells will form a segment. Additionally, for each pair of adjacent rows, these segments must intersect in at least one column to allow movement from one row to another.

This allows us to write the following dynamic programming: dpi,l,r is the number of ways to choose segments for the first i rows if the segment [l,r] is chosen in the last row. Unfortunately, there are too many states.

We can reduce the number of states as follows: instead of using the segment as the state, we will use the column in which we transition to the next row as the state. However, this may lead to counting the same configuration multiple times if two segments in adjacent rows intersect in more than one column. Therefore, we will consider that between each pair of adjacent rows, we transition in the leftmost column where the corresponding segments intersect; that is, if the segments in adjacent rows are [l1,r1] and [l2,r2], we can only transition between them in column j=min(l1,l2).

This allows us to write the following dynamic programming relation: dp′i,j,f is the number of ways to fill i rows if we want to transition to the next row in column j, and f is a flag that indicates whether j is the leftmost visited cell in the i-th row.

Now, for each row, there are only 2m states. This allows us to speed up the dynamic programming using binary matrix exponentiation: we will form the vector [dpi,1,0,dpi,1,1,dpi,2,0,dpi,2,1,…,dpi,m,0,dpi,m,1,si], where si is the sum of the dynamic programming values for the first i rows; then we will construct a matrix of size (2m+1)×(2m+1) that transforms this vector into the same vector for i+1. This matrix can be constructed "naively" in O(m4). Then we can exponentiate this matrix in O(m3logn) and compute the dynamic programming values significantly faster.

There is only one problem: the modulus in the problem is not fixed. This is actually to prevent solutions via Berlekamp-Massey, but one of the side effects is that the compiler cannot optimize operations modulo a non-constant modulus, and matrix multiplication works very slowly.

We can speed it up as follows: let's take the modulus as infrequently as possible during matrix multiplication. Let k=2m+1, and we want to multiply two matrices A and B of size k×k and write the result in matrix C. Then Ci,j=k∑x=1Ai,x⋅Bx,j, and this sum needs to be computed modulo. We can store this sum in an unsigned long long and take it modulo p every 16 operations (thanks to the fact that the product Ai,x and Bx,j does not exceed 1018).

There are other optimizations for matrix multiplication, but this one is sufficient. There are also options to create a matrix of size (m+1)×(m+1), which would also speed up the solution by 8 times, but that was not required.

Solution (Neon)
#include <bits/stdc++.h>
 
using namespace std;
 
#define forn(i, n) for (int i = 0; i < int(n); ++i)
 
const int K = 301;
 
using mat = vector<vector<int>>;
 
int n, m, mod, k;
 
int add(int x, int y) {
  x += y;
  if (x >= mod) x -= mod;
  return x;
}
 
int mul(int x, int y) {
  return x * 1LL * y % mod;
}
 
unsigned long long aux[K][K];
 
mat mul(mat a, mat b) {
  mat b2(k, vector<int>(k));
  forn(x, k) forn(y, k) b2[x][y] = b[y][x];
  mat c(k, vector<int>(k));
  forn(x, k) forn(y, k) {
    aux[x][y] = 0;
    forn(z, k) {
      aux[x][y] += a[x][z] * 1ll * b2[y][z];
      if((z & 15) == 15)
        aux[x][y] %= mod;
    }
    c[x][y] = aux[x][y] % mod;
  }
  return c;
}
 
mat binpow(mat a, int b) {
  mat c(k, vector<int>(k));
  forn(i, k) c[i][i] = 1;
  while (b) {
    if (b & 1) c = mul(c, a);
    a = mul(a, a);
    b >>= 1;
  }
  return c;
}
 
int main() {
  cin >> n >> m >> mod;
  k = 2 * m + 1;
  mat a(k, vector<int>(k));
  a[k - 1][k - 1] = 1;
  forn(i, m) forn(f, 2) {
    for (int l = f ? 0 : i; l <= i; ++l) {
      for (int r = i; r < m; ++r) {
        a[i * 2 + f][k - 1] = add(a[i * 2 + f][k - 1], 1);
        for (int j = l; j <= r; ++j) {
          a[i * 2 + f][j * 2 + (j == l)] = add(a[i * 2 + f][j * 2 + (j == l)], 1);
        }
      }
    }
  }
  auto res = binpow(a, n);
  cout << res[1][k - 1] << endl;
}

#define int long long

const int MOD = 998244353;

int power(int a, int b) {
    int res = 1;

        if (b & 1) res = res * a % MOD;
        a = a * a % MOD;
        b >>= 1;
    }
    return res;

    int c = 0, base = 0;
    for (int i = 0; i < n; i++) {
        c = (c + (i + 1) % MOD * inv(a[i]) % MOD) % MOD;
        base = (base + c) % MOD;
    }

    // Monotonic stack to find contribution ranges
    vector<int> L(n), R(n);
    stack<int> st;

    for (int i = 0; i < n; i++) {
        while (!st.empty() && a[st.top()] > a[i]) st.pop();
        L[i] = st.empty() ? -1 : st.top();
        st.push(i);
    }

    while (!st.empty()) st.pop();

    for (int i = n - 1; i >= 0; i--) {
        while (!st.empty() && a[st.top()] >= a[i]) st.pop();
        R[i] = st.empty() ? n : st.top();
        st.push(i);
    }

    // Count subarrays where each value is minimum
    map<int, int> freq;
    for (int i = 0; i < n; i++) {
        int cnt = (i - L[i]) * (R[i] - i);
        freq[a[i]] += cnt;
    }

    vector<pair<int, int>> res(freq.begin(), freq.end());
    int sz = res.size();

    // Prefix: processed contribution (denominator → 1 then numerator)
    vector<int> prefGain(sz);
    vector<int> prefCount(sz);

    int sumGain = 0;
    int sumCount = 0;

    for (int i = 0; i < sz; i++) {
        int x = res[i].first;
        int cnt = res[i].second;

        int val = ((-(x - 1) % MOD + MOD) % MOD 
                  + (x - 1) * inv(x) % MOD) % MOD;

        sumGain = (sumGain + val * cnt % MOD) % MOD;
        prefGain[i] = sumGain;

        sumCount += cnt;
        prefCount[i] = sumCount;
    }

    // Suffix: sum of cnt * inv(x)
    vector<int> suffInv(sz);
    int sumInv = 0;

    for (int i = sz - 1; i >= 0; i--) {
        int x = res[i].first;
        int cnt = res[i].second;

        sumInv = (sumInv + cnt * inv(x) % MOD) % MOD;
        suffInv[i] = sumInv;
    }

    // Total number of subarrays
    int totalSubarrays = n * (n + 1) / 2;

    // Process queries
    for (int qi = 0; qi < m; qi++) {
        int K = k[qi];

        if (K < res[0].first) {
            // All in k/x case
            int ans = (base + K % MOD * suffInv[0] % MOD) % MOD;
            cout << ans << '\n';
        }
        else if (K > res.back().first) {
            // All fully reduced + numerator
            int ans = (base 
                      + totalSubarrays % MOD * (K % MOD) % MOD 
                      + prefGain.back()) % MOD;
            cout << ans << '\n';
        }
        else {
            // Binary search split
            int l = 0, h = sz - 1, idx = 0;
            while (l <= h) {

                if (res[mid].first <= K) {
                    idx = mid;
                    l = mid + 1;
                } else h = mid - 1;

            int rightPart = 0;
            if (idx < sz - 1) {
                rightPart = (K % MOD) * suffInv[idx + 1] % MOD;
            }

            int ans = (base + leftPart + rightPart) % MOD;
            cout << ans << '\n';
        }
    }

    return 0;
}

Share a more natural solution of G with an (m+1)×(m+1) transition matrix.

First, the constraint that (1,1) should be covered by segments is equivalent to that of (n,m) by symmetry. Thus, let dpi,l,r denote the number of ways to choose segments for the first i rows where the segment for the i-th row is exactly [l,r], without considering the constraint of (1,1). Easy to write out the transition:

dpi,l,r=1+∑[a,b]∩[l,r]≠∅dpi−1,a,b

Note that [a,b]∩[l,r]=∅ iff either b<l or r<a. Let the prefix sum and the suffix sum as

prei,k=∑r≤kdpi,l,r,sufi,k=∑l≥kdpi,l,r

Then

dpi,l,r=1+prei−1,m−prei−1,l−1−sufi−1,r+1

Summing up dp by the definitions of pre and suf above gives the transitions between prei,sufi and prei−1,sufi−1. Flipping the grid horizontally gives the bijection between ways to choose segments of prei,k and that of sufi,m−k+1, showing that they are equal. Thus, only m+1 states (pre and a constant 1) should be maintained. The final answer is pren,m−pren,m−1.
