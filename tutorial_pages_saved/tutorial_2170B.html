2170A - Maximum Neighborhood

Idea: BledDest

Tutorial

2170A - Maximum Neighborhood

Because the matrix is small enough, you can iterate over each cell (i,j)(i,j) and its neighbors: (i−1,j)(i−1,j), (i+1,j)(i+1,j), (i,j−1)(i,j−1), and (i,j+1)(i,j+1) (don't forget to check if these cells exists). This allows you to calculate the cost for each cell.

However, there are solutions which don't need to consider the whole matrix. We can show that the only four cells we have to consider are (n,n)(n,n), (n−1,n)(n−1,n), (n,n−1)(n,n−1) and (n−1,n−1)(n−1,n−1) — the two rightmost cells in the last two rows. For every other cell, the cost of the cell below it or to the right of it will be strictly greater. This allows us to check only four cells.

In fact, we can also show that for n≥5n≥5, the cell (n−1,n−1)(n−1,n−1) is the optimal one, and its cost can be calculated using a simple formula. And the answers for n<5n<5 are given in the problem statement.

Solution 1 (Neon)
#include <bits/stdc++.h>
 
using namespace std;
 
int dx[] = {-1, 0, 1, 0};
int dy[] = {0, -1, 0, 1};
 
int main() {
    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        auto get = [&](int x, int y) {
            if (x >= 0 && x < n && y >= 0 && y < n) return x * n + y + 1;
            return 0;
        };
        int ans = 0;
        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < n; ++j) {
                int cur = get(i, j);
                for (int d = 0; d < 4; ++d) cur += get(i + dx[d], j + dy[d]);
                ans = max(ans, cur);
            }
        }
        cout << ans << '\n';
    }
}
Solution 2 (Neon)
#include <bits/stdc++.h>
 
using namespace std;
 
int main() {
  int t;
  cin >> t;
  while (t--) {
    int n;
    cin >> n;
    int ans = 1;
    if (n == 2) ans = 9;
    else if (n == 3 || n == 4) ans = 4 * n * n - n - 4;
    else if (n > 4) ans = 5 * n * n - 5 * n - 5;
    cout << ans << '\n';
  }
}

2170B - Addition on a Segment

Idea: FelixArg

Tutorial

2170B - Addition on a Segment

Let's count the number of non-zero elements in the array cc, as well as the sum of its elements ss. If s−c≥n−1s−c≥n−1, then the answer is cc; otherwise, the answer is the largest c′c′ such that 0<c′<c0<c′<c and s−c′≥n−1s−c′≥n−1.

Such a c′c′ can be found by simple enumeration or calculated using the formula:

c′=min(c,s−n+1)c′=min(c,s−n+1)

The first bound in the formula is quite obvious: if an element is equal to 00, then no operation has changed it, and therefore the longest segment could not be longer than cc.

The second bound is a bit trickier. Each operation is a segment of length L=ri−li+1L=ri−li+1 on which we add 11 to all its elements. In total, across all operations, we have added s=L1+L2+⋯+Lns=L1+L2+⋯+Ln. The longest segment among all operations is c′c′, then:

  • one segment has length c′c′;
  • the other segments have length of at least 11 according to the problem statement.

This means:

s≥c′+1+⋯+1s≥c′+1+⋯+1

s≥c′+(n−1)s≥c′+(n−1)

c′≤s−n+1c′≤s−n+1

Solution (FelixArg)
#include<bits/stdc++.h>
using namespace std;
 
#define int long long
 
void solve(){
    int n;
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++){
        cin >> a[i];
    }
 
    int sum = 0;
    int cnt_one = 0;
 
    for (int i = 0; i < n; i++){
        if (a[i] > 0){
            cnt_one++;
        }
        sum += a[i];
    }
 
    int sum2 = sum - cnt_one;
    int sub = n - 1 - sum2;
 
    cout << cnt_one - max(0ll, sub) << '\n';
}
 
signed main()
{

#endif

            - _clock_start).count() << "ms." << endl;

2170C - Quotient and Remainder

Idea: BledDest

Tutorial

2170C - Quotient and Remainder

Suppose we want to remove a pair of numbers qiqi and rjrj. We need to determine the minimum xx required for this. Since xmody=rixmody=ri, then y>riy>ri, or y≥rj+1y≥rj+1. We also know that ⌊xy⌋=qi⌊xy⌋=qi, thus x=y⋅⌊xy⌋+(xmody)=y⋅qi+rj≥(rj+1)⋅qi+rjx=y⋅⌊xy⌋+(xmody)=y⋅qi+rj≥(rj+1)⋅qi+rj.

Notice that if we take x=qi(rj+1)+rjx=qi(rj+1)+rj and y=rj+1y=rj+1, then this pair xx and yy indeed works for us. Therefore, we can remove the pair of elements (qi,rj)(qi,rj) if and only if qi(rj+1)+rj≤kqi(rj+1)+rj≤k.

Thus, our task is to form the largest number of pairs (qi,rj)(qi,rj) such that qi(rj+1)+rj=(qi+1)(rj+1)−1≤kqi(rj+1)+rj=(qi+1)(rj+1)−1≤k. Since if we can form cc pairs, we can also form c−1c−1 pairs, we can apply binary search on the answer.

Now our task is to determine whether we can form exactly cc pairs. Since our constraint is an upper limit on the product, it is always optimal to take the cc smallest qiqi and pair them with the cc smallest rjrj. Therefore, let's sort the arrays qq and rr in ascending order — then our task is to pair prefix of qq with prefix of rr of length cc.

Next, we need to show that it is always optimal to pair the smallest qiqi with the largest rjrj, that is, q1q1 with rcrc, q2q2 with rc−1rc−1, q3q3 with rc−2rc−2, and so on. To prove this, we will show that q1q1 is optimal to pair with rcrc.

By contradiction: suppose that in the optimal answer, the pairs (q1,ry)(q1,ry) and (qx,rc)(qx,rc) are formed. We know that q1≤qxq1≤qx and ry≤rcry≤rc. Also, since the pairs exist, it means that (q1+1)(ry+1)−1≤k(q1+1)(ry+1)−1≤k and (qx+1)(rc+1)−1≤k(qx+1)(rc+1)−1≤k. We will prove that we can swap the elements in the pairs and construct (q1,rc)(q1,rc) and (qx,ry)(qx,ry):

  • (q1,rc)(q1,rc): (q1+1)(rc+1)−1≤(qx+1)(rc+1)−1≤k(q1+1)(rc+1)−1≤(qx+1)(rc+1)−1≤k;
  • (qx,ry)(qx,ry): (qx+1)(ry+1)−1≤(qx+1)(rc+1)−1≤k(qx+1)(ry+1)−1≤(qx+1)(rc+1)−1≤k.
Thus, it is always optimal to pair q1q1 with rcrc and so on. Therefore, it is sufficient to check that if for all pairs (qi+1)(rc−i+1+1)−1≤k(qi+1)(rc−i+1+1)−1≤k, then cc pairs can be formed; otherwise, they cannot.

Thus, the solution works in O(nlogn)O(nlog⁡n) time due to sorting and binary search.

Solution (adedalic)
#include<bits/stdc++.h>
 
using namespace std;
 
#define fore(i, l, r) for(int i = int(l); i < int(r); i++)
#define sz(a) int((a).size())
 
typedef long long li;
 
template<class A> ostream& operator <<(ostream& out, const vector<A> &v) {
    fore(i, 0, sz(v)) {
        if(i) out << " ";
        out << v[i];
    }
    return out;
}
 
const int INF = int(1e9);
const li INF64 = li(1e18);
 
inline void solve() {
    int n; li k;
    cin >> n >> k;
    vector<int> q(n), r(n);
    fore (i, 0, n)
        cin >> q[i];
    fore (i, 0, n)
        cin >> r[i];
    
    sort(q.begin(), q.end());
    sort(r.begin(), r.end());
 
    auto check = [&](int cnt) {
        fore (i, 0, cnt) {
            if (li(q[i] + 1) * (r[cnt - 1 - i] + 1) - 1 > k)
                return false;
        }
        return true;
    };
 
    int lf = 0, rg = n + 1;
    while (rg - lf > 1) {
        int mid = (lf + rg) >> 1;
        if (check(mid))
            lf = mid;
        else
            rg = mid;
    }
    cout << lf << endl;
}
 
int main() {
#ifdef _DEBUG
    freopen("input.txt", "r", stdin);
    int tt = clock();
#endif
    ios_base::sync_with_stdio(false);
    cin.tie(0), cout.tie(0);
    cout << fixed << setprecision(15);
    
    int t = 1; cin >> t;
    while (t--) {
        solve();
        
#ifdef _DEBUG
        cerr << "TIME = " << clock() - tt << endl;
        tt = clock();
#endif
    }
    return 0;
}

2170D - Almost Roman

Idea: BledDest

Tutorial

2170D - Almost Roman

First, let's try to solve it with only one query. Which characters should we use, and where do we put them?

Greedy idea: let's use the maximum possible number of Is, then the maximum possible number of Vs, and only then use Xs. It's because if we replace any I with V, it increases the value by at least 33, and if we replace any V with X, it increases the value by exactly 55.

Okay, now we know which characters in which quantities we use. Now, let's choose their ordering in such a way that the number of pairs "IV" and "IX" is the maximum possible. V and X differ only in their costs, so let's replace every X (both in the original string and in characters we will use in the query) with V, and add the number of replaced characters times 55 to the answer.

Now imagine the following process. We replace every question mark with I and evaluate the string. But if we have too few Is in a query, we should change some of Is to Vs. When we change an I to a V, one of the following three things can happen:

  • we either increase the number of pairs "IV" by 11;
  • or we don't change it;
  • or we decrease it by 11.

Obviously, we want to make as many changes that increase the number of such pairs as we can, then as many changes that keep it as it is as we can, and only then apply changes that decrease it. Let's calculate two variables inc and same — the maximum number of changes of the first and the second type we can make.

Every segment of question marks is independent of each other. Suppose a segment is isolated; then the number of increasing changes is its length divided by 22 (because it is the maximum number of pairs we can have), and the number of changes of the second type is its length modulo 22 (because if it has even length, every character I we replace after making the pairs breaks one pair, and if it has odd length, we can transform an "IIV" pattern into an "IVV" pattern).

However, if there is an I before the segment or a V after it, they can also participate in forming pairs, so each of them increases its length by 11. V after the segment also reduces the number of increasing operations by 11, because we initially already have one pair "IV".

And that's about it: calculate the length of each segment of question marks, use it to calculate the values of inc and same. Then replace all question marks by Is to calculate the value of the string in the perfect case (when we have enough Is to replace), then use inc and same to find out how many replacements I →→ V increase or keep the number of pairs "IV". Processing queries also becomes easy if we know these values.

Solution (BledDest)
#include<bits/stdc++.h>

using namespace std;

int cost(char c)
{
    if(c == 'I') return 1;
    return 5;
}

int eval(string s)
{
    int n = s.size();
    int ans = 0;
    for(int i = 0; i < n; i++)
        if(s[i] == 'I' && i + 1 < n && s[i + 1] == 'V')
            ans--;
        else ans += cost(s[i]);
    return ans;    
}

void solve()
{
    int n, q;
    cin >> n >> q;
    string s;
    cin >> s;
    int ans = 0;
    for(int i = 0; i < n; i++)
    {
        if(s[i] == 'X')
        {
            s[i] = 'V';
            ans += 5;
        }
    }
    int inc = 0;
    int same = 0;
    int qcount = 0;
    for(int i = 0; i < n; i++)
    {
        if(s[i] != '?') continue;
        int r = i;
        while(r < n && s[r] == '?') r++;
        int len = r - i;
        if(i > 0 && s[i - 1] == 'I') len++;
        if(r < n && s[r] == 'V') 
        {
            inc--;
            len++;
        }
        inc += len / 2;
        same += len % 2;
        qcount += (r - i);
        i = r - 1;        
    }
    for(int i = 0; i < n; i++)
        if(s[i] == '?') s[i] = 'I';
    ans += eval(s);
    for(int i = 0; i < q; i++)
    {
        int a, b, c;
        cin >> a >> b >> c;
        int used_v = max(0, min(b, qcount - c));
        int used_x = max(0, qcount - c - b);
        int cur = ans;
        cur += used_v * 4 + used_x * 9;
        int rep = used_v + used_x;
        cur -= min(inc, rep) * 2;
        rep -= (inc + same);
        cur += max(0, rep) * 2;
        cout << cur << "\n";  
    }
}

int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    for(int i = 0; i < t; i++)
        solve();
}

2170E - Binary Strings and Blocks

Idea: Roms

Tutorial

2170E - Binary Strings and Blocks

First, let's try to simplify the definition of beautiful strings. If a string has only 11 block, then by removing that block, we will definitely end up with 00 blocks. However, if there is more than one block in the string, removing one of the outer blocks decreases the number of blocks by 11, while removing a block between two other blocks decreases the number of blocks by 22 (those two blocks merge). If there are 22 blocks in the string, we always end up with 11, and if there are more than 22, both options are available, so we can achieve any parity.

This actually means that a string is beautiful if and only if it contains more than one block (in other words, a string is not beautiful if and only if all its characters are the same).

Now our problem becomes the following: we need to count the number of binary strings in which both characters are present in every subsegment [li,ri][li,ri]. There are several approaches to solving this problem; in my opinion, the simplest is as follows:

Let dpidpi be the number of binary strings of length ii for which the characters in positions i−1i−1 and ii differ (that is, ii is the position of the last difference between the characters), and the constraints on all segments ending to the left of ii are satisfied. Let's try to understand how to calculate dpidpi knowing the previous values dpjdpj. If some segment [l,r][l,r] is contained within [j,i−1][j,i−1], then all characters in it will be the same. Therefore, we need to sum only those dpjdpj such that no segment [l,r][l,r] is fully contained within the segment [j,i−1][j,i−1].

Let's somehow find the minimum suitable jj for each position ii (obviously, all larger jj will also be suitable). After that, we will simply need to perform a range sum query, for which we can use either a Fenwick tree or prefix sums (which should be built in parallel to our dynamic programming).

How do we find such a position jj for ii? Let's find the maximum left boundary of the segment ending at each right boundary rr. However, this is not quite what we need, because the segment can end not only at exactly i−1i−1, but also before i−1i−1. But we can build prefix maxima for these values, so for each right boundary rr, we will know the maximum left boundary for all segments ending no later than rr, which is what we need.

Now let's understand how to calculate the answer based on this dynamic programming. In my opinion, the simplest way is to "extend" our string by 11 character and say that the character in position nn must differ from the previous one. Thus, dpndpn will contain the number of strings of length n+1n+1 for which all conditions are satisfied and the last two characters differ. By removing the last character, we obtain the number of strings of length nn for which all constraints are met. Therefore, the answer will be exactly dpndpn.

Thus, we have obtained a solution in O(nlogn)O(nlog⁡n) or O(n)O(n) depending on how we perform the range sum query.

Solution (BledDest)
#include<bits/stdc++.h>
 
using namespace std;
 
const int INF = 1e9;
const int MOD = 998244353;
 
int add(int x, int y)
{
    x += y;
    while(x >= MOD) x -= MOD;
    while(x < 0) x += MOD;
    return x;
}
 
void solve()
{
    int n, m;
    cin >> n >> m;
    vector<int> max_left(n + 1, -1);
    for(int i = 0; i < m; i++)
    {
        int l, r;
        cin >> l >> r;
        --l;
        --r;
        max_left[r] = max(max_left[r], l);
    }
    for(int i = 1; i <= n; i++)
        max_left[i] = max(max_left[i], max_left[i - 1]);
    vector<int> dp(n + 1);
    vector<int> prefsum(n + 2);
    dp[0] = 2;
    prefsum[1] = 2;
    for(int i = 1; i <= n; i++)
    {
        int j = max_left[i - 1] + 1;
        dp[i] = add(prefsum[i], -prefsum[j]);
        prefsum[i + 1] = add(prefsum[i], dp[i]);
    }
    cout << dp[n] << "\n";
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int t;
    cin >> t;
    for(int i = 0; i < t; i++) solve();
}

2170F - Build XOR on a Segment

Idea: BledDest

Tutorial

2170F - Build XOR on a Segment

This looks like a problem that requires linear algebra and XOR bases. However, we only need one fact from linear algebra: the size of the XOR base does not exceed the number of bits in the integers we are working with, so the answer to each query does not exceed B=12B=12.

Let's try to write a dynamic programming solution: dp[l,r,x,y]dp[l,r,x,y] — can we form the number xx in the interval [l,r)[l,r) using yy elements (the value of the dynamic programming will be 11 if it is possible, or 00 if it is not). For now, we won't go into detail about how this dynamic programming is implemented; we just need to understand that it exists and that we can compute the answer to each query as follows: we need to find the minimum yy such dp[l,r,x,y]=1dp[l,r,x,y]=1.

This dynamic programming has the following property: if dp[l,r,x,y]=1dp[l,r,x,y]=1, then dp[l−1,r,x,y]=1dp[l−1,r,x,y]=1, because we can use all the same elements in the segment [l−1,r][l−1,r]. Therefore, for each triple (r,x,y)(r,x,y), there exists a maximum value of ll such that dp[l,r,x,y]=1dp[l,r,x,y]=1. This means we can move this state of the dynamic programming to its value: we will obtain a new dynamic programming dp′[r,x,y]dp′[r,x,y] — the maximum ll such that we can form the number xx out of yy elements in the interval [l,r)[l,r). It is also easy to answer queries based on this: to answer a query, we need the minimum yy for which dp′[r,x,y]≥ldp′[r,x,y]≥l.

Let's try to understand how to compute this dynamic programming. We initialize dp[0,0,0]=∞dp[0,0,0]=∞, and for all other states we set −∞−∞. How do the values of dp change when transitioning from the ii-th layer to the (i+1)(i+1)-th layer? First, we can set that dp′i+1=dp′idpi+1′=dpi′ for all pairs (x,y)(x,y) (all subsets available with the right boundary ii remain available with the right boundary i+1i+1), and then we will try to "improve" it using the new element.

How can the new element aiai "improve" the values of the dynamic programming? For some (x,y)(x,y), there may be a way to form xx using yy elements with a more optimal left boundary. Firstly, the way to form aiai using 11 element will definitely improve, so we can assign dp′[i+1,ai,1]=idp′[i+1,ai,1]=i. Secondly, we can take some set that is reachable for the right boundary ii and add the element aiai to it; that is, we transition from dp′[i,x,y]dp′[i,x,y] to dp′[i+1,x⊕ai,y+1]dp′[i+1,x⊕ai,y+1]. The minimum index of the number that we will use in such a set will still be dp′[i,x,y]dp′[i,x,y], because the new number aiai has a higher index than all the previous numbers we have used. Therefore, if there is a way to form xx using yy elements with the right boundary ii and the left boundary dp′[i,x,y]dp′[i,x,y], we can form dp′[i+1,x⊕ai,y+1]dp′[i+1,x⊕ai,y+1] with the same left boundary.

This dynamic programming has a complexity of O(n⋅B⋅2B)O(n⋅B⋅2B), which is quite fast but does not fit into memory limits. To fix this, let's group all queries according to their right boundaries, and store only two (or even only one!) layer of dynamic programming, moving the right boundary and answering all queries for the current right boundary. This will reduce memory usage to O(B⋅2B)O(B⋅2B), which fits within the constraints.

Solution (BledDest)
#include<bits/stdc++.h>
 
using namespace std;
 
const int B = 13;
const int M = (1 << (B - 1));
const int INF = 1e9;
 
int dp[B][M];
 
void upd(int& a, int b)
{
    if(a < b) a = b;
}
 
int main()
{
    ios_base::sync_with_stdio(0);
    cin.tie(0);
    int n;
    cin >> n;
    vector<int> a(n);
    for(int i = 0; i < n; i++) cin >> a[i];
    vector<vector<int>> id(n);
    int q;
    cin >> q;
    vector<int> l(q), x(q);
    for(int i = 0; i < q; i++)
    {
        int r;
        cin >> l[i] >> r >> x[i];
        --l[i];
        --r;
        id[r].push_back(i);
    }
    
    for(int i = 0; i < B; i++)
        for(int j = 0; j < M; j++)
            dp[i][j] = -INF;
    dp[0][0] = INF;
    vector<int> ans(q, 0);
    for(int i = 0; i < n; i++)
    {
        int val = a[i];
        upd(dp[1][val], i);
        for(int j = 1; j < B - 1; j++)
        {
            for(int k = 1; k < M; k++)
            {
                upd(dp[j + 1][k ^ val], dp[j][k]);
            }
        }
        for(auto qr : id[i])
        {
            int cur = x[qr];
            for(int j = B - 1; j >= 0; j--)
                if(dp[j][cur] >= l[qr])
                    ans[qr] = j;
        }
    }
    
    for(int i = 0; i < q; i++)
        cout << ans[i] << " ";
    cout << endl;
}

    how do we know that we have to take the largest remainder and smallest quotient for the optimal answer

We choose the smallest for convenience.

    Not sure if this is helpful, but I struggled with that condition too and this is how I understand it.

    We need n actions exactly according to the problem statement. So we need to be able to perform n-1 more actions after doing our first c-length action.

    Intuitively, the smallest action we can do is adding 1 to a single element. Hence, given a particular array, we can at most perform S actions, where S is the sum of its elements.

    So then we need s-c >= n-1, where s-c is what we need after one c-length action, and n-1 is the actions we need to take.

    The wording might be a bit weird, but it's just referring to the net increase per item. For People who didn't understand Editorial for B:

here is the simple explanation: first, we need to find the number of non-zero components and call it cnt. and find the sum of all the elements in an array and call it sum.

Now, lets take an example: 0 0 1 1 4

to convert it to 0 0 0 0 0 in exactly n operations, we need to know the size of the very first operation lets call size of the first operation to be x, if we do:

0 0 [0 0 3]

then sum — 3*(-1) will be the sum after performing op1, we need to check if the value of new sum >= n-1 or not, if yes then its okay, else it won't be possible to perform remaining n-1 ops

so we need to find the range for the first operation such that sum >= n-1, now since one relation was

original sum — x >= n-1

then original sum — n + 1 >= x thus our final answer is min(x,number of non-zero components (cnt as we declared it earlier));

for reference, check this: https://codeforces.com/contest/2170/submission/351669568
