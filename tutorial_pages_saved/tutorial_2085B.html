2085A - Serval and String Theory

Hint 1

When ss is lexicographically equal to or greater than the reversal of ss, is it possible to make ss universal by performing the operations?

Hint 2

If possible, what is the minimum number of the operations required to make ss universal?

Solution

Case 1. If ss contains only one kind of letter, the answer will be NO.

Case 2. If ss is universal already, no operation is needed, and the answer will be YES.

Otherwise, ss falls into the following Case 3.

Case 3.1. When ss is not a palindrome, one can make ss universal in one operation by swapping the first pair of different letters of ss and its reversal.

s=reversal of s=st––ringssg––nirts}−→−−swapsgrintss=st_ringsreversal of s=sg_nirts}→swapsgrints

Case 3.2. When ss is a palindrome, swapping any two distinct letters will make ss no longer a palindrome. Note that the reversal of ss after the swap can be obtained by swapping the letters in the symmetric positions of the original ss. Since either ss or its reversal is universal, the original ss can be made universal in one operation.

s=le––v––el−→−−swap{s′=reversal of s′=lv––e––ellee––v––l}leevls=le_v_el→swap{s′=lv_e_elreversal of s′=lee_v_l}leevl

Thus, for Case 3, the answer will be NO if no operation can be performed; otherwise, the answer will be YES.

Code (Python)
getint = lambda: int(input())
getints = lambda: map(int, input().split())

def solve():
	n, k = getints()
	s = input().strip()
	if s < s[::-1] or (k >= 1 and min(s) != max(s)):
		print('YES')
	else:
		print('NO')

t = getint()
for _ in range(t):
	solve()

2085B - Serval and Final MEX

Hint 1

Before the last operation, what should the array be to obtain 00?

Hint 2

How to make all the elements in the array non-zero?

Solution

Note that before the last operation, all the integers in the array should be positive. Thus, we aim to turn the zeroes in aa into non-zeroes. To achieve this, we split the given array aa into two halves and perform the operations on the halves that contain zeroes. After that, there will be no zeroes in either of the two halves. The final 00 can be obtained by performing the last operation on the entire array.

[1,0,1,1––––––––,2,0,8,5]→[2,2,0,8,5––––––––]→[2,1––––]→[0][1,0,1,1_,2,0,8,5]→[2,2,0,8,5_]→[2,1_]→[0]
Another Hint

In fact, the final element can be 00 after the operations for any array of length 44. Can you solve the problem under the constraint of n=4n=4?

Another Solution

When n>4n>4, you can perform the operation on any subarray of length (n−3)(n−3). After the operation, the array aa will contain only 44 integers, which can be solved by brute force.

Code (Python)
getint = lambda: int(input())
getints = lambda: map(int, input().split())

def solve():
	n = getint()
	a = list(getints())
	op = []
	mid, r = n // 2, n
	if 0 in a[mid:]:
		op.append([mid + 1, n])
		r -= n - mid - 1
	if 0 in a[:mid]:
		op.append([1, mid])
		r -= mid - 1
	op.append([1, r])
	print(len(op))
	for o in op:
		print(*o)

t = getint()
for _ in range(t):
	solve()

2085C - Serval and The Formula

Hint 1

The formula (x+k)+(y+k)=(x+k)⊕(y+k)(x+k)+(y+k)=(x+k)⊕(y+k) is equivalent to (x+k)&(y+k)=0(x+k)&(y+k)=0, where && denotes the bitwise AND operation.

Hint 2

Note that a power of 22 shares no common bits with any positive integer less than it.

Solution

It can be shown that such an non-negative integer kk does not exist when x=yx=y.

When x≠yx≠y, one can show that k=2n−max(x,y)k=2n−max(x,y) is a possible answer, where 2n2n is a power of 22 that is sufficiently large.

Code (Python)
getint = lambda: int(input())
getints = lambda: map(int, input().split())

def solve():
	x, y = getints()
	if x == y:
		print(-1)
	else:
		print(2 ** 48 - max(x, y))

t = getint()
for _ in range(t):
	solve()

2085D - Serval and Kaitenzushi Buffet

Hint 1

Do not go for DP.

Hint 2

How many plates of sushi can be taken?

Hint 3

What are the constraints on the time of taking plates of sushi?

Solution

Note that the last ii-th time taking a plate of sushi should be no later than the (n−i⋅(k+1)+1)(n−i⋅(k+1)+1)-th minute. This constraint limits the time that a taking action can be performed. Therefore, we enumerate the nn minutes in chronological order, and greedily take the untaken plates delivered before with the greatest deliciousness when we reach a constraint, i.e., the current minute is the (n−i⋅(k+1)+1)(n−i⋅(k+1)+1)-th minute for a certain ii. The deliciousnesses of the untaken plates can be maintained by a heap. The total complexity is O(nlogn)O(nlog⁡n).

Code (C++)
#include <cstdio>
#include <queue>

using namespace std;

const int N = 2e5 + 5;

int n, k;
int d[N];
long long ans;

void solve() {
	scanf("%d%d", &n, &k);
	ans = 0;
	priority_queue <int> q;
	for (int i = 1; i <= n; i++) {
		scanf("%d", &d[i]);
		q.push(d[i]);
		if ((n - i + 1) % (k + 1) == 0) {
			ans += q.top();
			q.pop();
		}
	}
	printf("%lld\n", ans);
}

int main() {
	int t;
	scanf("%d", &t);
	while (t--)
		solve();
	return 0;
}

2085E - Serval and Modulo

Hint 1

What happens when an integer aiai mods kk?

Hint 2

Note that the difference between aiai and bibi is a multiple of kk.

Hint 3

The same holds for the difference between the sum of aiai and the sum of bibi.

Hint 4

Notice that d(n)≤2304d(n)≤2304 holds for all n≤1010n≤1010, where d(n)d(n) denotes the number of divisors of nn.

Solution

Note that if such a magic number kk exists, it will be a divisor of Δ=∑ai−∑biΔ=∑ai−∑bi when Δ≠0Δ≠0, or it can be any integer greater than all the aiai. Thus, we can check all the divisors of ΔΔ, resulting in a solution of O(∑(n⋅d(∑ai)+∑ai−−−−√))O(∑(n⋅d(∑ai)+∑ai)) time, where d(n)d(n) denotes the number of divisors of nn.

Code (C++)
#include <cstdio>

using namespace std;

const int N = 1e4 + 5;
const int C = 1e6 + 5;

int n;
int a[N], b[N];
int cnt[C];
long long s;

bool check(int k) {
	for (int i = 1; i <= n; i++)
		cnt[a[i] % k]++;
	for (int i = 1; i <= n; i++)
		if (--cnt[b[i]] < 0) {
			cnt[b[i]] = 0;
			for (int i = 1; i <= n; i++)
				cnt[a[i] % k] = 0;
			return 0;
		}
	return 1;
}

void solve() {
	scanf("%d", &n);
	s = 0;
	for (int i = 1; i <= n; i++) {
		scanf("%d", &a[i]);
		s += a[i];
	}
	for (int i = 1; i <= n; i++) {
		scanf("%d", &b[i]);
		s -= b[i];
	}
	if (s == 0) {
		printf("%d\n", check(0xc0ffee) ? 0xc0ffee : -1);
		return;
	}
	for (int i = 1; 1ll * i * i <= s; i++)
		if (s % i == 0) {
			if (check(i)) {
				printf("%d\n", i);
				return;
			}
			if (i < C && check(s / i)) {
				printf("%d\n", (int)(s / i));
				return;
			}
		}
	printf("-1\n");
}

int main() {
	int t;
	scanf("%d", &t);
	while (t--)
		solve();
	return 0;
}

2085F1 - Serval and Colorful Array (Easy Version)

Hint 1

Consider the integers in the colorful subarray after swapping.

Hint 2

How to gather these integers together, forming a colorful subarray, with the minimum number of swaps?

Hint 3

Consider the middle one among all the the elements in the colorful subarray.

Solution

Consider the elements in the colorful subarray after swapping. The first observation is as follows.

Observation. Mark the elements in the colorful subarray. Before swapping, to make these marked elements continuous, gathering them toward the middle one among them minimizes the number of swaps.

Proof. It is not optimal to swap two marked elements. Thus, the leftmost xx elements will move to their right, and the rightmost (k−x)(k−x) elements will move to their left. They will meet at some point between the xx-th element and the (x+1)(x+1)-th element. If x<k2x<k2, adjusting the meeting point to its right reduces (k−x)−x>0(k−x)−x>0 swaps. If x>k2x>k2, we can adjust it to its left and reduce the number of swaps similarly. Therefore, one can show that setting the meeting point to the middle of them is optimal through adjustments. □◻

From the observation we can also conclude that, the minimum number of swaps required to make the marked elements continuous is the difference between the sum of the distances from each marked element to the middle position before swapping and that sum after swapping, if the marked elements are swapped toward the middle position. Notice that the latter sum is a fixed constant only related to kk.

Focusing on the middle position of the colorful subarray, we enumerate all the positions in the array as possible middle positions. For each possible middle position pp, to form the colorful subarray whose middle position is pp, we should choose k2k2 distinct integers from its left and the other k2k2 from its right. The rounding of k2k2 will not affect the correctness as long as the sum of the rounded numbers is kk.

For each 1≤i≤k1≤i≤k, we should determine whether ii should be chosen from the left side of pp or the right side of pp. Let the distance from pp to the rightmost integer ii on its left be lili, and the distance for the right side be riri. We should choose either lili or riri for each ii under the constraint that both the number of chosen lili and that of riri are k2k2, minimizing the sum of the distances we choose. This can be done by first forcing to choose all the lili, and then choosing the smallest k2k2 elements of (ri−li)(ri−li) greedily.

In implementation, when enumerating the middle position pp, both lili and riri can be simply calculated in O(n)O(n) time, and then sort (ri−li)(ri−li) in O(nlogn)O(nlog⁡n) time. The total complexity is O(n2logn)O(n2log⁡n).

Code (C++)
#include <cstdio>
#include <algorithm>

using namespace std;

const int N = 3005;
const long long oo = 1e18;

int n, k;
int a[N], l[N], r[N], d[N];
long long ans;

void solve() {
	scanf("%d%d", &n, &k);
	for (int i = 1; i <= n; i++)
		scanf("%d", &a[i]);
	ans = oo;
	for (int i = 1; i <= n; i++) {
		for (int j = 1; j <= k; j++)
			l[j] = r[j] = n + 1;
		for (int j = i; j; j--)
			l[a[j]] = min(l[a[j]], i - j);
		for (int j = i; j <= n; j++)
			r[a[j]] = min(r[a[j]], j - i);
		long long s = 0;
		for (int j = 1; j <= k; j++) {
			s += l[j];
			d[j] = r[j] - l[j];
		}
		sort(d + 1, d + k + 1);
		for (int j = 1; j <= (k + 1) / 2; j++)
			s += d[j];
		ans = min(ans, s);
	}
	for (int i = 1; i <= k; i++)
		ans -= abs(i - (k + 1) / 2);
	printf("%lld\n", ans);
}

int main() {
	int T;
	scanf("%d", &T);
	while (T--)
		solve();
	return 0;
}

2085F2 - Serval and Colorful Array (Hard Version)

Hint 1

Is it possible to simplify the constraints?

Hint 2

Is it possible to remove the constraint that exactly k2k2 elements should be chosen from both sides of the enumerated position?

Solution

Optimizing the solution of F1 using (possibly, heavy) data structures results in a O(nlogn)O(nlog⁡n) solution.

Here we introduce a linear approach for F2 came up by Error_Yuan. In fact, the following proposition holds.

Proposition. The answer obtained is still correct without the constraint that exactly k2k2 elements should be chosen from both sides of the enumerated position.

Without the constraint, when enumerating the position pp, we can directly choose the smaller one between lili and riri for each 1≤i≤k1≤i≤k.

Proof. On the one hand, notice that the distance sum will not be smaller when pp is not the middle position among the chosen elements. On the other hand, consider the colorful subarray that produces the optimal answer, and mark the elements in it first. When pp enumerates to the middle position of the marked elements before swapping, the minimal distance sum without the constraint is no greater than that under the constraint. Therefore, all the considered candidates are no less than the real answer, and the real answer can be reached, completing the proof. □◻

In implementation, we consider the contribution to each position pp for each integer 1≤i≤k1≤i≤k. For a certain ii, it can be shown that the delta of the contribution will be one of −1−1, 00, or +1+1 when pp moves to an adjacent position. The delta of the contribution remains unchanged for most positions, and only changes either at the position where ii placed, or at the midpoint between two adjacent ii. By pre-calculating the changes of contribution deltas over all the ii and performing the prefix sum on the array twice, we can obtain the distance sums, resulting in an O(n)O(n) solution.

Code (C++)
#include <cstdio>
#include <algorithm>

using namespace std;

const int N = 4e5 + 5;
const long long oo = 1e18;

int n, k;
int a[N];
int last[N], dd[N];
long long s, d, ans;

void solve() {
	scanf("%d%d", &n, &k);
	for (int i = 1; i <= n; i++)
		last[i] = dd[i] = 0;
	s = 0;
	for (int i = 1; i <= n; i++) {
		scanf("%d", &a[i]);
		if (!last[a[i]])
			s += i - 1;
		else {
			int pre = last[a[i]];
			int mid = (pre + i) / 2;
			if ((pre + i) & 1) {
				dd[mid]--;
				dd[mid + 1]--;
			} else
				dd[mid] -= 2;
		}
		dd[i] += 2;
		last[a[i]] = i;
	}
	ans = oo;
	d = -k;
	for (int i = 1; i <= n; i++) {
		ans = min(ans, s);
		d += dd[i];
		s += d;
	}
	for (int i = 1; i <= k; i++)
		ans -= abs(i - (k + 1) / 2);
	printf("%lld\n", ans);
}

int main() {
	int T;
	scanf("%d", &T);
	while (T--)
		solve();
	return 0;
}

    aa ^ bb = aa + bb - 22 * (aa & bb) so the equation is aa + bb = aa + bb - 22 * (aa & bb) That means that (aa & bb) = 00 — (xx + kk) & (yy + kk) = 00

    We observe that a power of two is represented like that 10000...10000... and any number before it doesn't have the most significant bit on so their ANDing will always be 00. We want to add kk to both numbers such that the greater of them becomes a power of 22 and the other one becomes less than it.

    If both are equal then their ANDing after addition will never be 0. Otherwise, we can find the first nn such that 2n2n >= max(x,y)max(x,y) and make kk equal to 2n2n - max(x,y)max(x,y)

2085D - Serval and Kaitenzushi Buffet Could someone please help me understanding what is wrong with my approach or code.

Approach:

While processing the time(0-index based) from n — 2(second last) to 0, I am trying to find out if I take the plate at time = i, then what is maximum sum of deliciousness of plates I can take at time > i, such that taking and consuming those plates (k + 1 for each plate) takes time <= n — 1 — (i + k), as this is the time left when I take plate i -> i + k time consumed. For that i am using a segment tree, in which I am storing for time = 0 to n, the (maximum sum of deliciousness, time) for each range processed by far.

Code:

#include <iostream>
#include <vector>

using namespace std;

struct Node {
    long long val;
    int len;
    Node () : val(0), len(0) {};
    Node (const long long& val, const int& len) : val(val), len(len) {};
    void output () {
        cout << val << ' ' << len << ' ';
    }
    const bool operator< (const Node& oth) const {
        return val == oth.val ? len > oth.len : val < oth.val;
    }
};

template <typename T>
struct SegTree {
    int n;
    vector<T> segTree;

    SegTree (const int& n) : n(n), segTree(n) {}

    T comp (const T& a, const T& b) {
        return max(a, b);
    }

    void update (int node, int l1, int r1, T& val, int tl, int tr) {
        if (l1 > r1) return;
        if (l1 == tl && r1 == tr) {
            segTree[node] = max(segTree[node], val);
            return;
        }
        int m = tl + (tr - tl) / 2, n1 = node + 1, n2 = node + 2 * (m + 1 - tl);
        update(n1, l1, min(r1, m), val, tl, m);
        update(n2, max(l1, m + 1), r1, val, m + 1, tr);
        segTree[node] = comp(segTree[n1], segTree[n2]);
    }

    T query (int node, int l1, int r1, int tl, int tr) {
        if (l1 > r1) return Node();
        if (l1 == tl && r1 == tr) return segTree[node];
        int m = tl + (tr - tl) / 2, n1 = node + 1, n2 = node + 2 * (m + 1 - tl);
        return comp(query(n1, l1, min(r1, m), tl, m), query(n2, max(l1, m + 1), r1, m + 1, tr));
    }

    void output () {
        for (int i = 0; i < n; i++) cout << segTree[i].val << ' '; // cout << segTree[i].output() << ' ';
        cout << "\n";
    }
};

int main () {
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    cout.tie(NULL);

    short t;
    cin >> t;
    for (short tc = 1; tc <= t; tc++) {
        int n, m, k;
        cin >> n >> k;
        int d[n];
        m = 2 * (n + 1) - 1;
        SegTree<Node> segTree(m);
        for (int i = 0; i < n; i++) cin >> d[i];
        for (int i = n - 2; i >= 0; i--) {
            if (i + k < n) {
                int j1 = n - 1 - i - k, j2 = n - 1 - i - 2 * k - 1;
                Node val1 = segTree.query(0, 0, j1, 0, n), val2;
                if (i + 2 * k + 1 < n) val2 = segTree.query(0, 0, j2, 0, n);
                val1.val += d[i];
                val1.len += k + 1;
                segTree.update(0, val1.len, val1.len, val1, 0, n);
                if (i + 2 * k + 1 < n) {
                    val2.val += d[i];
                    val2.len += k + 1;
                    segTree.update(0, val2.len, val2.len, val2, 0, n);
                }
            }
        }
        cout << segTree.query(0, 0, n, 0, n).val << "\n";
    }

    return 0;
}
