2208A - Bingo Candies

Idea by Mindeveloped, prepared by Mindeveloped

Hint 1

It is always possible to rearrange the candies, unless candies of one specific color appears too often.

Solution

Let xx be the color with the highest number of candies. In case of ties, xx could be any of the candidates. Denote the number of candies with color xx as aa.

Conclusion: The answer to the problem is YES if and only if there are no more than n2−nn2−n candies of color xx; — in other words, a≤n2−na≤n2−n holds.

Proof of necessity: If there are more than n2−nn2−n candies of color xx, then there are less than nn candies not of color xx. Since there are nn rows, at least one row will have no candies not of color xx; therefore, such a row will be filled with candies of color xx, violating the restrictions.

Proof of sufficiency: If there are no more than n2−nn2−n candies of color xx, we can always construct a solution with the method below.

First, if a<na<n holds, then we can arrange the candies arbitrarily because no color has enough candies to fill a row or a column.

Otherwise, n≤a≤n2−nn≤a≤n2−n holds.

We can put candies of color xx in a1,1,a2,2,…,an,na1,1,a2,2,…,an,n. This will take nn candies of color xx. Next, we choose nn candies not of color xx arbitrarily. We will put them at a1,2,a2,3,…,an−1,n,an,1a1,2,a2,3,…,an−1,n,an,1. Lastly, we arrange other candies that have not yet been arranged arbitrarily. Since in the first step we put a candy of color xx in each row and column, and in the second step we put a candy not of color xx in each row and column, each row and column contains at least candies of two colors, satisfying the constraints.

2208B - Cyclists

Idea by Mindeveloped, prepared by Mindeveloped

Solution

For the sake of simplicity, let's assume all cards have different costs. If multiple cards have the same cost, we break ties by their initial position in the deck (the earlier card is considered to have a lower cost for the purpose of the strategy).

The optimal strategy: If the win-condition is among the first kk cards, then we play it. Otherwise, we play the card in the first kk places with the lowest cost. Proof of optimality is shown below.

Lemma: If at any time, the win-condition is placed at position x>kx>k, the cost used before the win-condition is played again is at least the sum of the costs of x−kx−k cards with the lowest costs out of all the x−1x−1 cards placed before the win-condition. Our strategy reaches this minimum value.

Proof:

Since playing any card in front of the win-condition moves the win-condition 1 place toward the front, we need to play x−kx−k normal cards before the win-condition is playable.

Before the win-condition moves to position kk and becomes playable, every card placed before the win-condition can be played at most once, and every card placed at the back of the win-condition cannot be played. Therefore, the cost used is at least the sum of the first x−kx−k lowest costs of the cards initially placed before the win-condition. We mark the x−kx−k cards placed before the win-condition with the lowest costs.

Among all the x−1x−1 cards placed before the win-condition, exactly k−1k−1 cards are not marked, so whenever before the win-condition is played the first time, at least one marked card will be in the first kk places. Therefore, our strategy will always play a marked card in the first x−kx−k plays since their costs are always lower than not marked ones. Since each marked card can be only played for at most once, after x−kx−k plays we will play each marked card once, which achieves the lower bound described above.

Therefore, the lemma is proven.

Now, let AA be the sum of the costs of p−kp−k cards with the lowest costs placed before the win-condition. Let BB be the sum of the costs of n−kn−k cards with the lowest costs, without considering the win-condition.

By the above lemma, the cost spent before playing the win-condition the first time is at least AA. After it is played each time, it always gets placed at position nn, so the cost spent before playing the win-condition another time is at least BB. Our strategy achieves the minimum.

Therefore, the cost used to play the win-condition any non-negative number of times is minimized, which proves the optimality of the strategy.

The answer is ⌊m−A−apB+ap⌋+1⌊m−A−apB+ap⌋+1 if m−A−ap≥0m−A−ap≥0. Otherwise, the answer is 00.

The value of AA and BB can be calculated with a priority queue in O(nlogn)O(nlog⁡n). Therefore, the overall time complexity is O(nlogn)O(nlog⁡n).

Alternatively, you may simulate the strategy above directly; the time complexity will be O(nm)O(nm), which is also sufficient to pass.

2208C - Stamina and Tasks

Idea by tybbs, prepared by tybbs

Solution

Notice that the value of SS does not affect your decision. This meant for integer i∈[1,n−1]i∈[1,n−1] the decision of tasks [1,i][1,i] does not affect the decision of tasks [i+1,n][i+1,n].

Let fifi be the answer for considering tasks [i,n][i,n] only and assuming that S=1S=1. It holds that fi=max(fi+1,fi+1⋅(1−pi100)+ci)fi=max(fi+1,fi+1⋅(1−pi100)+ci), and f1f1 is the answer needed.

Code
#include<bits/stdc++.h>
#define ld long double
using namespace std;
const int N=1e5+5;
ld c[N],p[N];
void solve(){
	int n;
	cin>>n;
	for(int i=1;i<=n;i++){
		int cv,pv;
		cin>>cv>>pv;
		c[i]=cv,p[i]=pv/100.0;
	}
	ld ans=0;
	for(int i=n;i>=1;i--){
		ans=max(ans,ans*(1-p[i])+c[i]);
	}
	cout<<fixed<<setprecision(10)<<ans<<"\n";
}
int main(){
	ios::sync_with_stdio(0);
	cin.tie(0);
	int T;
	cin>>T;
	for(int i=1;i<=T;i++) solve();
}

2208D2 - Tree Orientation (Hard Version)

Idea by tybbs, prepared by tybbs

Hint 1

If a solution exists, it must be unique.

Hint 2

Assume a solution exists, construct the graph accordingly, and verify afterward that it satisfies all given conditions.

Hint 3

Consider the necessary and sufficient condition for the existence of a directed edge u→vu→v.

Answer to Hint 3 (Solution for D1)

Two conditions must hold:
1. Node vv must be reachable from uu.
2. There must be no intermediate node ww such that uu can reach ww and ww can reach vv.

Implementing this directly yields an O(n3)O(n3) solution, which is sufficient for D1. One can further optimize it to O(n3ω)O(n3ω) using bitsets, though this is not required for this subtask.

Hint 4

Let RuRu denote the set of nodes reachable from uu, excluding uu itself.

Let susu denote the number of nodes reachable from uu. Among all nodes in RuRu, let vv be the one with the largest svsv. What can we infer?

Answer to Hint 4

It can be proven that the edge u→vu→v must exist in the solution.

Solution

We iteratively apply the idea from Hint 4:

  • Find the node v∈Ruv∈Ru with the largest svsv, and add the edge u→vu→v.
  • Then remove all nodes in RvRv from RuRu, since they are already reachable from uu indirectly via vv, and therefore cannot have a direct edge from uu.
  • Repeat this process until RuRu becomes empty. At that point, all outgoing edges from uu have been determined.

Since the final graph is a tree with exactly n−1n−1 edges, the total number of such removal operations across all nodes is O(n)O(n). Each operation involves scanning or updating a set of size at most nn, leading to an overall time complexity of O(n2)O(n2).

Code
#include <bits/stdc++.h>
using namespace std;
constexpr int N = 1e4+5;
int n, id[N], cnt[N];
bool vs[N], e[N][N];
struct Dsu {
	int fa[N];
	void init(int n) {
		for (int i = 1; i <= n; i++) fa[i] = i;
	}
	int find(int x) {
		if (x == fa[x]) return x;
		return fa[x] = find(fa[x]);
	}
	void merge(int x, int y) {
		fa[find(x)] = find(y);
	}
} dsu;
bool reach[N];
vector<int> tree[N];
void search(int u) {
	reach[u] = 1;
	for (int v : tree[u]) {
		search(v);
	}
}
void solve(int tid) {
	cin >> n;
	for (int i = 1; i <= n; i++) cnt[i] = 0, tree[i].clear();
	for (int u = 1; u <= n; u++) {
		id[u] = u;
		string s;
		cin >> s;
		for (int v = 1; v <= n; v++) {
			e[u][v] = s[v-1]-'0';
			cnt[u] += e[u][v];
		}
	}
	sort(id + 1, id + n + 1, [&](int u, int v) {
		return cnt[u] > cnt[v];
	});
	vector<pair<int, int >> edges;
	for (int u = 1; u <= n; u++) {
		for(int i = 1; i <= n; i++) {
			vs[i] = false;
		}
		vs[u] = true;
		for (int i = 1; i <= n; i++) {
			int v = id[i];
			if (!vs[v] && e[u][v]) {
				edges.push_back({u, v});
				if(edges.size() >= n) {
					cout << "No" << "\n";
					return ;
				}
				for (int w = 1; w <= n; w++) {
					if (e[v][w]) {
						vs[w] = true;
					}
				}
			}
		}
	}
	if (edges.size() != n - 1) {
		cout << "No" << "\n";
		return ;
	}
	sort(edges.begin(), edges.end());
	dsu.init(n);
	for (auto [u, v] : edges) {
		dsu.merge(u, v);
		tree[u].push_back(v);
	}
	int flag = 1;
	for (int i = 1; i <= n; i++) flag &= (dsu.find(i) == dsu.find(1));
	if (!flag) {
		cout << "No" << "\n";
		return ;
	}
	for (int i = 1; i <= n; i++) {
		for (int j = 1; j <= n; j++) reach[j] = 0;
		search(i);
		for (int j = 1; j <= n; j++) {
			if (reach[j] != e[i][j]) {
				cout <<  "No" << "\n";
				return ;
			}
		}
	}
	cout<< "Yes\n";
	for (auto [u, v] : edges) {
		cout << u << ' ' << v << '\n';
	}
}
int main() {
	ios::sync_with_stdio(0), cin.tie(0);
	int T;
	cin >> T;
	for (int i = 1; i <= T; i++) solve(i);
	return 0;
}

2208E - Counting Cute Arrays

Idea by tybbs and Mini_PEKKA, prepared by tybbs

Hint 1

Find a straightforward criterion to determine whether an array is "cute".

Answer to Hint 1

First, consider a necessary condition: If Xi≥iXi≥i for any ii, the array XX cannot be cute.

It is well-known that the structure of XX relates to a monotonic stack. We can verify whether an array XX of length nn is cute using the following simulation procedure:

  1. Maintain a stack SS, initialized with a single sentinel element 00.
  2. Iterate ii from 11 to nn:
    • Check if the value XiXi exists in SS.
    • If XiXi is not in SS, then XX is not a cute array.
    • If XiXi is in SS, pop elements from SS until XiXi becomes the top element. Then, push ii onto the stack.
  3. If the loop completes without violating the conditions above, XX is a cute array.
Hint 2

Solve the problem for the following special case: For all ii, XiXi is either −1−1 or i−1i−1.

Answer to Hint 2

We can use dynamic programming to solve this special case. Let fi,kfi,k denote the number of ways to fill the −1−1s in the prefix X1…XiX1…Xi such that the array remains potentially cute and the current stack size is kk.

The transition is straightforward: * If Xi=−1Xi=−1, we can pop an arbitrary number of elements. Thus, fi,k=∑ij=k−1fi−1,jfi,k=∑j=k−1ifi−1,j. * If Xi=i−1Xi=i−1, we cannot pop any elements (the previous index must be the nearest greater element). Thus, fi,k=fi−1,k−1fi,k=fi−1,k−1.

Hint 3

Analyze the failure condition of the stack procedure further.

Answer to Hint 3

If XiXi is not found in SS during the simulation, it implies that XiXi was previously popped from the stack while processing some earlier index jj. This indicates the existence of a pair (j,i)(j,i) such that Xj<Xi<j<iXj<Xi<j<i.

Consequently, an array XX is cute if and only if no such pair (j,i)(j,i) exists that satisfies this condition.

Solution

First, check if all fixed values (where Xi,Xj≠−1Xi,Xj≠−1) satisfy the condition that no pair (j,i)(j,i) with j<ij<i exists such that Xj<Xi<j<iXj<Xi<j<i. Additionally, ensure Xi<iXi<i for all ii. If these conditions are not met, the answer is 00.

Consider each fixed Xi≠−1Xi≠−1 as defining an interval [Xi,i][Xi,i]. By sorting these intervals by length from smallest to largest, we can process them hierarchically. Once all XkXk within a specific interval are determined, the interval effectively behaves like the base case [i−1,i][i−1,i] mentioned in Hint 2.

This reduction transforms the general problem into the special case solved in Hint 2. We can compute the final answer using the DP approach with prefix sum optimization, resulting in an overall time complexity of O(n2)O(n2).

Code
#include<bits/stdc++.h>
using namespace std;
const int N=5e3+5,mod=998244353;
inline void add(int &x,int v){ x=(x+v)%mod; }
int T,n,ans,a[N],id[N],mx[N],f[N],sf[N];
void calc(int l,int r){
	int tp=0,cnt=0;
	f[0]=1,sf[0]=1;
	while(1){
		if(mx[l]==0) l++,tp=0;
		else l=mx[l],tp=1;
		if(l>r) break;
		cnt++;
		if(tp==0){
			for(int i=1;i<=cnt;i++) f[i]=sf[i-1];
			f[0]=0;
		}
		else{
			for(int i=cnt;i>=1;i--) f[i]=f[i-1];
			f[0]=0;
		}
		sf[cnt+1]=0;
		for(int i=cnt;i>=0;i--) sf[i]=(sf[i+1]+f[i])%mod;
	}
	ans=1ll*ans*sf[0]%mod;
}
void solve(){
	cin>>n;
	for(int i=0;i<=n;i++) mx[i]=0;
	for(int i=1;i<=n;i++){
		cin>>a[i];
	}
	for(int i=1;i<=n;i++){
		if(a[i]>=i) return void(cout<<0<<"\n");
	}
	for(int i=1;i<=n;i++){
		for(int j=i+1;j<=n;j++){
			if(a[i]!=-1 && a[j]!=-1){
				if(a[i]<a[j] && a[j]<i) return void(cout<<0<<"\n");
			}
		}
	}
	int cnt=0;
	for(int i=1;i<=n;i++){
		if(a[i]!=-1) id[++cnt]=i;
	}
	sort(id+1,id+cnt+1,[](int x,int y){ return x-a[x]<y-a[y]; });
	ans=1;
	for(int i=1;i<=cnt;i++){
		calc(a[id[i]],id[i]-1);
		mx[a[id[i]]]=id[i];
	}
	calc(0,n);
	cout<<ans<<"\n";
}
int main(){
	ios::sync_with_stdio(0);
	cin.tie(0);
	cin>>T;
	for(int i=1;i<=T;i++) solve();
}
