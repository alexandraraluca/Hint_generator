1744A - Number Replacement

Let's note that if ai=ajai=aj, then sisi must be equal to sjsj, since we must change the same value to the same letter. If we check this for all pairs of ii and jj and find no such contradictions, then the answer is "YES", otherwise "NO".

We got the solution for O(n2)O(n2) for one test case.

1744B - Even-Odd Increments

Let sumsum — the initial sum in the array, cnt0cnt0 — the number of even numbers, cnt1cnt1 — the number of odd numbers.

Let's see how these values ​​change with each action. In fact, we can consider four main options:

  • Add an even number xx to all even numbers. Then sumsum will increase by x⋅cnt0x⋅cnt0, and the number of even and odd numbers will remain the same.
  • Add an odd number xx to all even numbers. Then sumsum will increase by x⋅cnt0x⋅cnt0, the number of even numbers will become 00, all numbers will become odd, so cnt1=ncnt1=n.
  • Add an even number xx to all odd numbers. Then sumsum will increase by x⋅cnt1x⋅cnt1, and the number of even and odd numbers will remain the same.
  • Add an odd number xx to all odd numbers. Then sumsum will increase by x⋅cnt1x⋅cnt1, the number of odd numbers will become 00, all numbers will become even, so cnt0=ncnt0=n.

1744C - Traffic Light

Let's note that for each second of color cc in the traffic light, we need to find the rightmost green time, and then find the largest distance between color cc and the nearest green. Also, let's not forget that traffic light states are cyclical.

To get rid of cyclicity, you can write the string ss twice and for each cell of color cc from the first half, find the nearest green color (thus we solved the problem with cyclicity). And now we can just follow this line from right to left and maintain the index of the last occurrence of green. If we encounter color cc, then we try to update our answer ans=max(ans,last−i)ans=max(ans,last−i), where ansans is our answer, lastlast is the nearest time that green was on color, ii — current time.

1744D - Divisibility by 2^n

Let's notice that if we multiply the numbers a1⋅a2⋅…⋅ana1⋅a2⋅…⋅an, then the power of two of the product is equal to the sum of the powers of two in each of the numbers.

Let's calculate the initial sum of the powers of twos in the product. This can be done as follows: let's take the element aiai and divide it by 22 as long as we can, while remembering to increase our counter by the number of occurrences of a power of two.

Now let's move on to operations and note that choosing the index ii will increase the degree of occurrence of two by a fixed number (that is, it does not matter when to apply this operation). Choosing an index ii will increment the counter by a number xx such that ii is divisible by 2x2x but not by 2x+12x+1 — you can find this xx, again , by dividing by 22 while we can.

Since we want to minimize the number of operations used, at each moment of time we will try to use an index that increases the counter by the largest number. To do this, it is enough to sort the indices by this index of increase and take them greedily from the largest increase to the smallest.

We get the solution in O(nlogn+nlogA)O(nlog⁡n+nlog⁡A).

1744E2 - Divisible Numbers (hard version)

Let's look at the slow solution first, which will lead us to the full one. Let's iterate over the number xx from a+1a+1 to cc.

Given a number xx, we want to find a yy from b+1b+1 to dd such that x⋅yx⋅y is divisible by a⋅ba⋅b. Since x⋅yx⋅y must be divisible by a⋅ba⋅b, the following conclusion can be drawn: yy must be divisible by a⋅bgcd(a⋅b,x)a⋅bgcd(a⋅b,x). Let's denote this number as ss. Now our task is to check if there is a multiple of ss between b+1b+1 and dd. The problem can be solved in many ways, you can simply consider the largest multiple of ss that does not exceed dd — it is equal to ⌊ds⌋⋅s⌊ds⌋⋅s We compare this number with b+1b+1, and if it matches, then we have found a suitable pair.

Now let's note that we do not need to iterate over all the values ​​of xx, because from the number xx we are only interested in gcd(x,a⋅b)gcd(x,a⋅b) — and this is one of the divisors of a⋅ba⋅b!

Even though the product a⋅ba⋅b can be large, we can still consider all divisors of this number, since aa and bb themselves are up to 109109. Let's find the divisors of aa and bb separately, then notice that any divisor of a⋅ba⋅b — is a′⋅b′a′⋅b′, where a′a′ — is some divisor aa, and b′b′ — is some divisor bb.

Let's calculate the running time. We need to factorize the numbers aa and bb into prime factors, this can be done in O(a−−√)O(a). Next, you need to iterate over pairs of divisors aa and bb. Recall the estimate for the number of divisors of a number (https://oeis.org/A066150): the number xx up to 109109 has no more than 13441344 divisors. Therefore, we can sort through the pairs for 1344213442. We learned how to find the optimal yy for O(1)O(1). We get that for each test case we have learned to solve the problem in O(a−−√+(13442))O(a+(13442)).

1744F - MEX vs MED

Let's learn how to count the number of subsegments where mex>midmex>mid for a fixed value of mexmex.

Let's understand on which subsegments mexmex has such a value. We understand that the numbers 0,1,…,mex−10,1,…,mex−1 should be in this subsegment, the number mexmex — should not, and then — does not matter . Let ℓℓ — be the index of the left-most occurrence of the numbers 0,1,…,mex−10,1,…,mex−1, and rr — the right-most occurrence. Also, let pospos "be the index of the number mexmex. If ℓ<pos<rℓ<pos<r, then there is no subsegment with the given mexmex, otherwise let's think about how many numbers can be in a subsegment.

It is stated that if the length of a segment is ≤2⋅mex≤2⋅mex, then mex>midmex>mid on it, otherwise it is not true. Indeed, if the length of a segment is ≤2⋅mex≤2⋅mex, then simply by the definition of the median, it will be among the first mexmex numbers. On the other hand, if the length of the segment becomes longer, then according to the same definition of the median, it will no longer be among the first mexmex numbers, which are equal to 0,1,…,mex−10,1,…,mex−1.

It turns out that we need to count the number of subsegments of length ≤2⋅mex≤2⋅mex that contain the subsegment ℓ…rℓ…r inside themselves, but not pospos. This, of course, can be done with large formulas and the like, but one more useful fact can be noticed!

If we iterate over mexmex in ascending order, and at the same time maintain ℓℓ and rr, then let's see what happens with a fixed mexmex and a position of pospos (for convenience, let's assume that pos<ℓpos<ℓ, since the second case is similar). In fact, you can simply iterate over the left border of our segment from pos+1pos+1 to ℓℓ, and then use a simpler formula to calculate the number of good subsegments with such fixed values.

Why can this left boundary be iterated for each mexmex if pos<ℓpos<ℓ (correspondingly, if it were pos>rpos>r, then we would iterate over the right boundary) ? Let's remember that after this step, ℓℓ becomes equal to pospos. That is, if we iterate over an element, then we move the corresponding border. And we can only move it O(n)O(n) times.

    This is not something that you "prove". It is something that you intuitively know. When you unpack this statement it doesn't actually say much but the math makes it kind of convoluted so I'll explain it. To have some x*y be divisible by a*b, the set of prime divisors in a*b should be a subset of prime divisors in x*y. In other words, for every prime divisor in a*b, that prime divisor should also exist in x*y. Now if you think about it, gcd(a*b, x) simply means the set of prime divisors in x, which are also prime divisors in a*b. This means that the y that you choose only has to contain the prime divisors that is in a*b but not in x. This is given by the expression u gave ((x*y)/gcd(a*b, x)).
