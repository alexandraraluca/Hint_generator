2199A - Game

Idea: BledDest, preparation: BledDest

Tutorial

2199A - Game

There are several ways to solve this problem.

The most straightforward approach is to iterate through how many points Alice and Bob will score in the last round and check who will win.

A simpler solution is to consider only the scenario where Alice scores 00 points and Bob scores kk points in the last round.

However, it is actually sufficient to check that Alice is ahead of Bob by at least kk points based on the results of the first two rounds. If her lead is less than kk, then Bob will surpass her if he scores kk points while she scores 00. But if Alice has scored at least kk points more than Bob in total over the two rounds, then she has won the first two rounds; and even if Bob surpasses her by kk points in the third round, Alice will still win.

Thus, it is enough to verify that a1+a2−b1−b2≥ka1+a2−b1−b2≥k.

Solution (BledDest)
fun main() {
    val t = readLine()!!.toInt()
    repeat(t) {
        val k = readLine()!!.toInt()
        val (a1, b1) = readLine()!!.split(" ").map { it.toInt() }
        val (a2, b2) = readLine()!!.split(" ").map { it.toInt() }
        println(if (a1 - b1 + a2 - b2 >= k) "NO" else "YES")
    }
}

2199B - Two Towers

Idea: BledDest, preparation: Neon

Tutorial

2199B - Two Towers

Using only the first type of operations, we can achieve the goal in (c−a)+(d−b)(c−a)+(d−b) actions. However, we can "save" 11 action for each use of the second type of operation. Thus, it is advantageous to use as many operations of the second type as possible. The earliest we can start these operations is when both towers are max(a,b)max(a,b) high, and the latest is when both are min(c,d)−1min(c,d)−1 high. Therefore, the number of operations of the second type is at most min(c,d)−max(a,b)min(c,d)−max(a,b) (or 00 if this value is negative). Then the minimum number of operations is (c−a)+(d−b)−max(0,min(c,d)−max(a,b))(c−a)+(d−b)−max(0,min(c,d)−max(a,b)).

Solution (Neon)
fun main() = repeat(readln().toInt()) {
    var (a, b, c, d) = readln().split(" ").map { it.toInt() }
    if (a > b) a = b.also { b = a }
    if (c > d) c = d.also { d = c }
    println(d - a - maxOf(0, b - c))
}

2199C - Minesweeper

Idea: BledDest, preparation: Neon

Tutorial

2199C - Minesweeper

Let there be two mines, one in column ii and the other in column jj. If |i−j|≤2|i−j|≤2 (that is, the mines are in the same column, in adjacent columns, or one column apart), then at least one cell will be adjacent to both of these mines; this means that there must also be a mine in that cell. It can be shown that this will ultimately lead to the entire field being filled with mines. Therefore, in the answer, the column numbers where the mines are located must differ by at least 33, and there cannot be two mines in the same column.

If one mine is placed in a column, it will increase the number of free cells adjacent to the mines by 55, except when it is in the first or last column (in which case the number of free cells adjacent to the mines will only increase by 33). Moreover, no more than two mines can increase the number of these cells by 33 (one in the first column and one in the last column). Therefore, for an answer to exist, kk must be expressible as k=5mk=5m, k=5m+3k=5m+3, or k=5m+6k=5m+6, where mm is a non-negative integer. If such a representation exists for some kk, it is unique (due to different remainders modulo 55). We will find such a representation, placing mm blocks of 33 columns, each containing a mine, and if necessary, we will add 22 columns to the left and/or right with a mine in the first/last column.

The case k=1k=1 needs to be considered separately

Solution (Neon)
fun printAns(n : Int, pos: MutableList<Int>) {
    if (n == 0) {
        println("NO")
        return
    }
    println("YES")
    println(n)
    val a = CharArray(n) { '.' }
    println(a.joinToString(""))
    for (i in pos) {
        a[i] = '*'
    }
    println(a.joinToString(""))
}

fun main() = repeat(readln().toInt()) {
    var k = readln().toInt()
    if (k == 1) {
        printAns(1, mutableListOf(0))
        return@repeat
    }
    var n = 0
    val pos = mutableListOf<Int>()
    if (k % 5 == 1 || k % 5 == 3) {
        n += 2
        pos.add(0)
        k -= 3
    }
    while (k >= 5) {
        n += 3
        pos.add(n - 2)
        k -= 5
    }
    if (k % 5 == 3) {
        n += 2
        pos.add(n - 1)
        k -= 3
    }
    printAns(if (k == 0) n else 0, pos)
}

2199D - Two Arrays

Idea: Roms, preparation: Roms

Tutorial

2199D - Two Arrays

The first observation is as follows: if it is possible to make two arrays identical, then we can subsequently apply the same operations to them until their sizes become equal to 11. Therefore, let's check if there exists some number xx to which both arrays can be transformed.

Let's try to understand what this xx could be for each array. If the array already has length 11, then the only element of this array is the only possible value for xx.

Otherwise, let's determine which elements of the array can be xx and which cannot. The first and last elements cannot be xx, since in order to reduce the size to 11, each element must participate in at least one operation; but as soon as we use the minimum or maximum in an operation, they will be removed.

Let's prove that any other element, except for the first or last, can be the only remaining one. Suppose we want to leave only the ii-th element in the end. If there are an equal number of elements on the left and right, we can use all elements in one operation, and the ii-th will remain as a result. If, for example, there are more elements on the left, we can choose any 33 elements from the left and remove 22 of them. Since the parity of the number of elements on the left and right of ii is the same, we can perform this operation until the number of elements on the left and right of ii is equal. Thus, we can leave the ii-th element as the last one.

Now we know how to determine for each array all the elements that can be the only remaining ones. Let a′a′ be the array of all elements to which we can transform aa (b′b′ is defined similarly). To solve the problem, it is now sufficient to check that there is at least one element that belongs to both a′a′ and b′b′. We can iterate through all elements of a′a′ and check using binary search if this element exists in b′b′; or we can convert both arrays to a Set and call the intersect method.

Solution (Roms)
import java.util.StringTokenizer

fun main() {
    val t = readLine()!!.toInt()
    repeat(t) {
        val st = StringTokenizer(readLine())
        st.nextToken()
        st.nextToken()

        val a = readLine()!!.split(" ").map { it.toInt() }
        val b = readLine()!!.split(" ").map { it.toInt() }

        val aTrimmed =
            if (a.size > 1) a.subList(1, a.size - 1) else a
        val bTrimmed =
            if (b.size > 1) b.subList(1, b.size - 1) else b

        val ok = aTrimmed.toSet().intersect(bTrimmed.toSet()).isNotEmpty()
        println(if (ok) "Yes" else "No")
    }
}

2199E - Supersequence

Idea: BledDest, preparation: Neon

Tutorial

2199E - Supersequence

Let's try to understand what the array bb might look like. To construct the shortest array in which aa appears as a subsequence, we can use the following process:

  • set b1=a1b1=a1;
  • add as few elements as possible so that the next one can be a2a2. If a1<a2a1<a2, then each subsequent element should be greater than the previous one by 11. If a1>a2a1>a2, then each subsequent element should be less than the previous one by 11. If a1=a2a1=a2, we can place either a1+1a1+1 or a1−1a1−1 between them;
  • similarly go from a2a2 to a3a3;
  • and so on.

Thus, the number of elements that will be between the occurrences of aiai and ai+1ai+1 in the array bb can be easily determined: if they are equal, then there is 11 element between them; otherwise, there are |ai−ai+1|−1|ai−ai+1|−1 elements between them. This allows us to determine the position of each element aiai in the array bb.

Consider the elements between the occurrences of aiai and ai+1ai+1. If these two numbers are equal, we cannot uniquely determine the element between them — it could be either ai+1ai+1 or ai−1ai−1. However, if they are different, all elements between them are uniquely determined, and knowing aiai, ai+1ai+1, and the index of the element in the block between them makes it easy to reconstruct the element itself.

Thus, to answer each query, we need to know which block the element at position xixi belongs to (or, possibly, whether it is an occurrence of one of the elements in the array aa). If we list the starting positions of each block in the array, we can determine which block xixi belongs to using binary search. This gives us a solution in O(n+qlogn)O(n+qlog⁡n).

Solution (Neon)
import kotlin.math.abs

fun main() {
    val (n, q) = readln().split(" ").map { it.toInt() }
    val a = readln().split(" ").map { it.toInt() }
    val xs = readln().split(" ").map { it.toLong() }
    val bl = IntArray(2 * n - 1) { 1 }
    for (i in 1 until n) {
        if (a[i] != a[i - 1]) bl[2 * i - 1] = abs(a[i] - a[i - 1]) - 1
    }
    val s = bl.scan(0L) { x, y -> x + y }
    
    fun getVal(x : Long) : Int {
        if (x > s.last()) return -1
        val i = s.binarySearch { if (it < x) -1 else 1 }.inv() - 1
        val j = i / 2
        if (i % 2 == 0) return a[j]
        if (a[j] == a[j + 1]) return 0
        val d = (a[j + 1] - a[j]) / abs(a[j + 1] - a[j])
        return a[j] + d * (x - s[i]).toInt()
    }
    
    println(xs.map { getVal(it) }.joinToString(" "))
}

2199F - Self-Produced Sequences

Idea: BledDest, preparation: Neon

Tutorial

2199F - Self-Produced Sequences

First, let's understand what self-produced sequences look like. Consider the first non-zero element; the sum of the elements to the right should be equal to this element (since the sum on the left is 00). Similarly, for the last non-zero element, the sum of the elements to the left of it should be equal to this element (since the sum on the right is 00). This implies that the only possible case when both of the above conditions are met is when there are exactly 22 non-zero elements in the sequence and they are equal. Now, let's determine where the zero elements can be placed. Clearly, zeros cannot be between non-zero elements because both the left and right sums are greater than 00. However, they can be in any other positions, either before or after the non-zero elements. Thus, we have determined that self-produced sequences have the form: [0,…,0][0,…,0] or [0,…,0,x,x,0,…,0][0,…,0,x,x,0,…,0], where x>0x>0.

Next, let's understand how to calculate the number of such subsequences in the array aa. Let prefiprefi be the number of 00s before position ii, and sufisufi be the number of 00s after position ii. Sequences consisting solely of zeros are straightforward to count: there are 2tot2tot (where tottot is the total number of 00s) such sequences. For sequences with two non-zero elements, fix two indices ii and jj such that i<ji<j and ai=ajai=aj. There are prefiprefi zeros to the left of ii, so the total number of ways to place these zeros is 2prefi2prefi. Similarly, for sufjsufj zeros to the right, there are 2sufj2sufj ways. Therefore, for a fixed pair (i,j)(i,j), the number of subsequences is 2prefi2sufj2prefi2sufj.

However, iterating through all pairs of (i,j)(i,j) would be inefficient. Instead, let's iterate over jj maintaining sxsx — the sum of 2prefi2prefi for all i<ji<j where ai=xai=x. For the current jj, we can increase the answer by saj2sufjsaj2sufj if aj≠0aj≠0. Then, increment sajsaj by 2prefj2prefj. Note that since the array values can reach up to 109109, use an associative array (like MutableMap in Kotlin) or coordinate compression to store ss efficiently.

Solution (Neon)
const val MOD = 998244353;

fun add(x: Int, y: Int): Int {
    return ((x + y) % MOD + MOD) % MOD
}

fun mul(x: Int, y: Int): Int {
    return (x.toLong() * y % MOD).toInt()
}

fun main() = repeat(readln().toInt()) {
    val n = readln().toInt()
    val a = readln().split(" ").map { it.toInt() }
    val pw = generateSequence(1, { mul(it, 2) }).take(n + 1).toList()
    val cnt = a.scan(0) { x, y -> x + if (y == 0) 1 else 0 }
    val sum = mutableMapOf<Int, Int>()
    var ans = pw[cnt[n]]
    for (i in 0 until n) {
        if (a[i] == 0) continue
        val cur = sum.getOrDefault(a[i], 0)
        ans = add(ans, mul(cur, pw[cnt[n] - cnt[i]]))
        sum[a[i]] = add(cur, pw[cnt[i]]) 
    }
    println(ans)
}

2199G - Jammer

Idea: adedalic, preparation: adedalic

Tutorial

2199G - Jammer

Without loss of generality, let's assume that n≤mn≤m. Otherwise, we can simply "reflect" the field and the problem will remain unchanged.

Let's pay attention to two "special" paths of the robot: on one path, the robot will first go up to the limit and then to the right, while on the other path, it will go to the right to the limit and then up. It is not difficult to prove that our jammer will definitely intercept the robot if and only if it intercepts it on both these special paths.

Let's understand where to place the jammer so that it intercepts the robot on the path (0,0)−(0,m)−(n,m)(0,0)−(0,m)−(n,m). For this, we need to place the jammer at a position (x,y)(x,y) such that either x≤rx≤r (the robot will be intercepted on the segment (0,0)−(0,m)(0,0)−(0,m)), or y≥m−ry≥m−r (the robot will be intercepted on the segment (0,m)−(n,m)(0,m)−(n,m)). Similarly, for the path (0,0)−(n,0)−(n,m)(0,0)−(n,0)−(n,m), we will get either y≤ry≤r or x≥n−rx≥n−r.

Then, let's iterate the coordinate xx of the jammer from 00 to nn and determine all suitable coordinates yy. What restrictions does the first path impose? If x≤rx≤r, then all y≥⌊r2−x2−−−−−−√⌋+1y≥⌊r2−x2⌋+1 are suitable for us (we simply cannot choose points that are too close to (0,0)(0,0)). Otherwise (if x>rx>r), only y≥m−ry≥m−r is suitable for us. Thus, the first path imposes a lower bound on possible yy-s.

Similarly, if n−x≤rn−x≤r, then all y≤m−1−⌊r2−(n−x)2−−−−−−−−−−−√⌋y≤m−1−⌊r2−(n−x)2⌋ are suitable for us (meaning they are sufficiently far from (n,m)(n,m)). Otherwise (if n−x>rn−x>r), only y≤ry≤r is suitable for us. Thus, the second path imposes an upper bound on possible yy-s. Therefore, suitable yy-s form some segment, and their number depends only on the length of that segment.

As a result, for each xx in [0,n][0,n], we can count all yy in constant time. Thus, the complexity of the solution is O(t⋅n)O(t⋅n) (or O(tmin(n,m))O(tmin(n,m)) in general). Since n⋅m≤109n⋅m≤109 and n≤mn≤m, it follows that n≤109−−−√n≤109.

Solution (adedalic)
import kotlin.math.*

fun main() {
    fun isqrt(a : Long) : Long {
        var s = max(0L, sqrt(a.toDouble()).toLong() - 2)
        while ((s + 1) * (s + 1) <= a)
            s++
        return s
    }
    repeat(readln().toInt()) {
        var (n, m, r) = readln().split(' ').map { it.toLong() }
        if (n > m)
            n = m.also { m = n }
        var ans = 0L
        for (i in 0..n) {
            val lf = if (i <= r) 1 + isqrt(r * r - i * i) else max(0L, m - r)
            val rg = if (n - i <= r) m - 1 - isqrt(r * r - (n - i) * (n - i)) else min(m, r)
            ans += max(0L, rg - lf + 1)
        }
        println(ans)
    }
}

2199H - Sum of MEX

Idea: BledDest, preparation: BledDest

Tutorial

2199H - Sum of MEX

To begin, let's try to solve the problem when all elements are equal to −1−1. We will calculate ansx,yansx,y — the number of arrays of size yy whose MEX is equal to xx. For all x,y≤kx,y≤k, this can be done in O(k3)O(k3): for each xx, we will separately calculate the dynamic programming table dpi,jdpi,j — the number of arrays of size ii with elements from 00 to nn, which contain jj distinct numbers from 00 to x−1x−1, but do not contain the number xx. Then, ansx,yansx,y is equal to dpi,xdpi,x for the given value of xx.

How do we compute this dynamic programming? Clearly, dp0,0=1dp0,0=1. From each state dpi,xdpi,x, there are two transitions:

  • either we add an element from 00 to x−1x−1, which was not present before — there are x−jx−j ways to choose it;
  • or we add an element greater than xx or one of the previously encountered elements — there are n−(x−j)n−(x−j) ways to choose it.

Now let's understand what to do with the elements from 00 to nn that are present in the array aa. How do we calculate the answer for a prefix of length ii? We will write down all the numbers from 00 to nn that were not among the first ii elements of aa. Let these numbers be x0,x1,…,xmx0,x1,…,xm in increasing order. Let there be cc elements equal to −1−1 in the prefix of length ii.

I claim that the number of arrays with MEX equal to xjxj, which can be obtained from the prefix of length ii, is exactly equal to ansj,cansj,c: there will always be exactly xj−jxj−j distinct elements present in the prefix of length ii in the array aa, so among the numbers that all −1−1 will be transformed into, there must necessarily be jj remaining numbers from 00 to xj−1xj−1, and jj must not be present.

Moreover, we are only interested in the first (c+1)(c+1) values from x0,x1,…,xmx0,x1,…,xm. We can maintain them using a mutableSet, DSU, or some other data structures. This allows us to process each prefix of the array aa in c+lognc+log⁡n time.

Overall complexity: O(k3+n⋅(k+logn))O(k3+n⋅(k+log⁡n)), where kk is the number of elements equal to −1−1.

Solution (BledDest)
import java.util.*

const val MOD = 998244353

fun add(x: Int, y: Int): Int {
    var result = x + y
    while (result >= MOD) result -= MOD
    while (result < 0) result += MOD
    return result
}

fun mul(x: Int, y: Int): Int {
    return ((x.toLong() * y) % MOD).toInt()
}

fun solve() {
    val scanner = Scanner(System.`in`)
    val n = scanner.nextInt()
    val a = IntArray(n) { scanner.nextInt() }
    var k = 0
    for (i in 0 until n) if (a[i] == -1) k++
    val ans = Array(k + 1) { IntArray(k + 1) }
    
    for (mex in 0..k) {
        val dp = Array(k + 1) { IntArray(mex + 1) }
        dp[0][0] = 1
        for (i in 0 until k) {
            for (j in 0..mex) {
                if (dp[i][j] == 0) continue
                val inc = mex - j
                val same = n - inc
                if (inc > 0) {
                    dp[i + 1][j + 1] = add(dp[i + 1][j + 1], mul(inc, dp[i][j]))
                }
                dp[i + 1][j] = add(dp[i + 1][j], mul(same, dp[i][j]))
            }
        }
        for (i in 0..k) {
            ans[mex][i] = dp[i][mex]
        }
    }

    val unusedFirst = mutableSetOf<Int>()
    val unusedAll = mutableSetOf<Int>()
    for (i in 0..n) unusedAll.add(i)
    var cnt = 0
    for (i in 0 until n) {
        if (a[i] == -1) {
            cnt++
        } else {
            unusedFirst.remove(a[i])
            unusedAll.remove(a[i])
        }
        while (unusedFirst.size <= cnt) {
            unusedFirst.add(unusedAll.first())
            unusedAll.remove(unusedAll.first())
        }
        val unused = unusedFirst.toList()
        var sum = 0
        for (j in 0..cnt) {
            sum = add(sum, mul(ans[j][cnt], unused[j]))
        }
        print("$sum ")
    }
}

fun main() {
    val t = 1
    repeat(t) {
        solve()
    }
}

2199I - Strange Process

Idea: BledDest, preparation: BledDest

Tutorial

2199I - Strange Process

Let f(x)f(x) denote the size of the factorization of xx (the number of primes in the factorization, considering that some primes may have different powers). For example, f(60)=4f(60)=4.

We also denote the minimum number of operations required to transform the number xx into the number yy as d(x,y)d(x,y). The operations we consider are multiplication or division by a prime number. We can calculate d(x,y)d(x,y) as follows: compute g=gcd(x,y)g=gcd(x,y), then d(x,y)=f(x)+f(y)−2⋅f(g)d(x,y)=f(x)+f(y)−2⋅f(g).

Let's try to understand how to determine if a certain array bb is achievable. If two non-zero elements bibi and bjbj (i<ji<j) are equal to each other, then it should be possible to transform cici into cjcj in no more than j−ij−i operations; in other words, d(ci,cj)≤j−id(ci,cj)≤j−i.

This allows us to write a dynamic programming state of the form dpj,i1,i2,…,indpj,i1,i2,…,in, where jj is the number of initial elements of bb that we have considered, and i1,i2,…,ini1,i2,…,in are the indices of the last elements of bb that were equal to 1,2,…,n1,2,…,n, respectively. Obviously, this dynamic programming has too many states. To reduce the number of states, we will use the following optimizations:

  • The maximum value of d(x,y)d(x,y) is 88. This means that the values i1,i2,…,ini1,i2,…,in that do not exceed (j−8)(j−8) are not of interest to us (they do not impose any restrictions on where the corresponding number will appear next in the array bb). We are only interested in values from (j−7)(j−7) to (j−1)(j−1);
  • If we change all occurrences of one number from 11 to nn to another and vice versa in an achievable array bb, it will remain achievable. This means that the order of i1,i2,…,ini1,i2,…,in is not of interest to us.

We can transform the dynamic programming state into dp′j,maskdpj,mask′, where maskmask is a binary mask of 77 bits describing which of the values j−1,j−2,…,j−7j−1,j−2,…,j−7 occur among i1,i2,…,ini1,i2,…,in. In other words, the kk-th bit of the mask describes whether there exists an ii such that bj−k=ibj−k=i, and this is the last occurrence of ii among the first jj elements in the array bb.

Transitions in this dynamic programming are done quite trivially:

  • Either we assign bj=0bj=0 (this can always be done);
  • Or we take one of those values that were not among the last 77 elements (the number of ways to choose such a value is n−popcnt(mask)n−popcnt(mask), where popcnt(mask)popcnt(mask) is the number of set bits in the mask);
  • Or we take one of those values that were among the last 77 elements. We can iterate through when it was, check that the corresponding bit in the mask is 11, and that we have enough operations to transform cj−kcj−k into cjcj.

Thus, we obtain a solution in O(mB2B)O(mB2B), where BB is the number of last elements of the sequence that we are interested in (with B=7B=7 being sufficient)

Solution (BledDest)
import kotlin.math.*
import java.util.*

const val MOD = 998244353
const val K = 7
const val A = 50

fun add(x: Int, y: Int): Int {
    var result = x + y
    while (result >= MOD) result -= MOD
    while (result < 0) result += MOD
    return result
}

fun mul(x: Int, y: Int): Int {
    return (x.toLong() * y % MOD).toInt()
}

fun gcd(x: Int, y: Int): Int {
    return if (x == 0) y else gcd(y % x, x)
}

fun solve() {
    val scanner = Scanner(System.`in`)
    val n = scanner.nextInt()
    val m = scanner.nextInt()
    val c = IntArray(m) { scanner.nextInt() }
    val dist = Array(A + 1) { IntArray(A + 1) }
    
    for (i in 1..A) {
        for (j in 1..A) {
            var diff = i * j / gcd(i, j) / gcd(i, j)
            for (k in 2..A) {
                while (diff % k == 0) {
                    diff /= k
                    dist[i][j]++
                }
            }
        }
    }

    val dp = IntArray(1 shl K)
    dp[0] = 1
    for (i in 0 until m) {
        val cur = c[i]
        val ndp = IntArray(1 shl K)
        for (j in 0 until (1 shl K)) {
            var popcount = 0
            val shifted = (j shl 1) and ((1 shl K) - 1)
            ndp[shifted] = add(ndp[shifted], dp[j])
            for (k in 0 until K) {
                if ((j shr k) and 1 == 1) popcount++
            }
            val unused = n - popcount
            if (unused > 0 && i >= dist[1][cur] - 1) {
                ndp[shifted + 1] = add(ndp[shifted + 1], mul(unused, dp[j]))
            }
            for (k in 0 until K) {
                if ((j shr k) and 1 == 0) continue
                val idx = i - k - 1
                if (idx >= 0 && dist[c[idx]][cur] > k + 1) continue
                var new_mask = shifted + 1
                if (k != K - 1) new_mask -= 1 shl (k + 1)
                ndp[new_mask] = add(ndp[new_mask], dp[j])
            }
        }
        for (k in dp.indices) {
            dp[k] = ndp[k]
        }
    }
    var ans = 0
    for (x in dp) ans = add(ans, x)
    println(ans)
}

fun main() {
    val t = 1
    repeat(t) {
        solve()
    }
}
