1487A - Arena

Idea: BledDest

Tutorial

1487A - Arena

If for some hero ii, no other hero is weaker than ii, then the ii-th hero cannot win any fights and is not a possible winner. Otherwise, the hero ii is a possible winner — he may fight the weakest hero 100500100500 times and be declared the winner.

So the solution to the problem is calculating the number of minimum elements in the array aa, since all other elements denote possible winners of the tournament.

Solution (Neon)
#include <bits/stdc++.h>

using namespace std;

int main() {
  int t;
  cin >> t;
  while (t--) {
    int n;
    cin >> n;
    vector<int> a(n);
    for (int& x : a) cin >> x;
    cout << n - count(a.begin(), a.end(), *min_element(a.begin(), a.end())) << endl;
  }
}

1487B - Cat Cycle

Idea: adedalic

Tutorial

1487B - Cat Cycle

If nn is even, then each hour A and B are on the spots with different parity, so they will never meet. Otherwise, let's look closely what happens.

At the start, A in nn and B in 11. But since we can form a cycle from spots then it means that nn and 11 in reality are neighbors. After that, A and B (starting from neighboring positions) just go in opposite directions and meet each other in the opposite spot after exactly ⌊n2⌋⌊n2⌋ steps.

After meeting B "jumps over" A making 11 extra step and the situation become practically the same: A and B are neighbors and move in the opposite direction.

In other words, each f=⌊n2⌋f=⌊n2⌋ steps B makes one extra step, so the answer (if both kk and spots are 00-indexed) is

(k+(nmod2)⋅⌊kf⌋)modn(k+(nmod2)⋅⌊kf⌋)modn

Solution (adedalic)
fun main() {
    repeat(readLine()!!.toInt()) {
        var (n, k) = readLine()!!.split(' ').map { it.toInt() }
        k--
        val floor = n / 2
        println((k + (n % 2) * k / floor) % n + 1)
    }
}

1487C - Minimum Ties

Idea: BledDest

Tutorial

1487C - Minimum Ties

If nn is odd, then we can solve the problem without any ties: each team should win exactly ⌊n2⌋⌊n2⌋ matches and lose the same number of matches. Finding which matches each team wins and which matches each team loses can be done with some graph algorithms (like Eulerian cycles or circulations), or with a simple construction: place all teams in a circle in any order, and let the ii-th team win against the next ⌊n2⌋⌊n2⌋ teams after it in the circle, and lose to all other teams.

Unfortunately, if nn is even, we need to use some ties since the total sum of scores over all teams is exactly 3n(n−1)23n(n−1)2 when there are no ties, and this number is not divisible by nn when nn is even. Each tie reduces the total sum by 11, and the minimum number of ties to make 3n(n−1)2−t3n(n−1)2−t divisible by nn is t=n2t=n2 (since 3n(n−1)2modn=n23n(n−1)2modn=n2). So, if we find an answer with exactly n2n2 ties, it is optimal. And it's easy to find one: once again, place all teams in a circle in any order; make the ii-th team win against n−22n−22 next teams in the circle, lose against n−22n−22 previous teams in the circle, and tie with the opposite team in the circle.

Solution (BledDest)
#include<bits/stdc++.h>

using namespace std;

int main()
{
    int t;
    cin >> t;
    for(int i = 0; i < t; i++)
    {
        int n;
        cin >> n;
        if(n % 2 == 1)
        {
            for(int i = 0; i < n; i++)
                for(int j = i + 1; j < n; j++)
                    if(j - i <= n / 2)
                        cout << 1 << " ";
                    else
                        cout << -1 << " ";
            cout << endl;
        }
        else
        {
            for(int i = 0; i < n; i++)
                for(int j = i + 1; j < n; j++)
                    if(j - i < n / 2)
                        cout << 1 << " ";
                    else if(j - i == n / 2)
                        cout << 0 << " ";
                    else
                        cout << -1 << " ";
            cout << endl;
        }
    }
}

1487D - Pythagorean Triples

Idea: Neon

Tutorial

1487D - Pythagorean Triples

We have to find the number of triples (a,b,ca,b,c) such that equations a2+b2=c2a2+b2=c2 and a2−b=ca2−b=c are satisfied. Let's subtract one equation from another and get that b2+b=c2−c⟹b(b+1)=c(c−1)b2+b=c2−c⟹b(b+1)=c(c−1). So we know that c=b+1c=b+1 and after substituting, we get that a2=2b+1a2=2b+1. We can see that there is only one correct value of bb (and cc) for every odd value of aa (greater than 11). So we can iterate over the value of aa and check that the corresponding value of cc doesn't exceed nn. This solution works in O(n−−√)O(n) because a≈c√≤n−−√a≈c≤n, but you can also solve it in O(1)O(1).

Solution (Neon)
#include <bits/stdc++.h>

using namespace std;

int main() {
  int t;
  cin >> t;
  while (t--) {
    int n;
    cin >> n;
    int ans = 0;
    for (int i = 3; i * i <= 2 * n - 1; i += 2)
      ++ans;
    cout << ans << '\n';
  }
}

1487E - Cheap Dinner

Idea: BledDest

Tutorial

1487E - Cheap Dinner

The main solution is dynamic programming: let dpidpi for every possible dish ii be the minimum cost to assemble a prefix of a dinner ending with the dish ii (here, ii can be a dish of any type: first course, second course, drink, or dessert). Then, the answer to the problem is the minimum value among all desserts.

The number of transitions in this dynamic programming is too big, since, for example, when transitioning from first courses to second courses, we need to check O(n1n2O(n1n2) options. To speed this up, we need some sort of data structure built over the values of dpidpi for all first courses ii that allows to recalculate dpjdpj for a second course jj quickly. There are two main approaches to this:

  • build any version of RMQ over the values of dynamic programming for the first courses. Then, when we want to calculate the answer for some second course jj, sort all types of first courses which don't go well with it, and make several RMQ queries to find the minimum value over all non-forbidden first courses;
  • store all values of dpidpi in a data structure that supports adding an element, deleting an element, and finding the minimum element (this DS should allow duplicate elements as well). When we want to calculate the answer for some second course jj, remove all values of dpidpi corresponding to the first courses that don't go well with it from the data structure, query the minimum in it, and insert the removed elements back.

The same approach can be used to advance from second courses to drinks and from drinks to desserts (you can even use the same code in a for-loop with 33 iterations, so the resulting solution is actually short and simple).

Solution (BledDest)
#include<bits/stdc++.h>

using namespace std;


int main()
{   
    vector<int> ns(4);
    for(int i = 0; i < 4; i++)
        scanf("%d", &ns[i]);
    vector<vector<int>> cs(4);
    for(int i = 0; i < 4; i++)
    {
        cs[i].resize(ns[i]);
        for(int j = 0; j < ns[i]; j++)
            scanf("%d", &cs[i][j]);
    }
    vector<vector<vector<int>>> bad(3);
    for(int i = 0; i < 3; i++)
    {
        bad[i].resize(ns[i + 1]);
        int m;
        scanf("%d", &m);
        for(int j = 0; j < m; j++)
        {
            int x, y;
            scanf("%d %d", &x, &y);
            x--;
            y--;
            bad[i][y].push_back(x);
        }
    }                   
    vector<vector<int>> dp(4);
    dp[0] = cs[0];
    for(int i = 0; i < 3; i++)
    {
        dp[i + 1].resize(ns[i + 1]);                
        multiset<int> s;
        for(int j = 0; j < ns[i]; j++)
            s.insert(dp[i][j]);
        for(int j = 0; j < ns[i + 1]; j++)
        {
            for(auto k : bad[i][j])
                s.erase(s.find(dp[i][k]));
            if(s.empty())
                dp[i + 1][j] = int(4e8 + 43);
            else
                dp[i + 1][j] = *s.begin() + cs[i + 1][j];
            for(auto k : bad[i][j])
                s.insert(dp[i][k]);       
        }
    }
    int ans = *min_element(dp[3].begin(), dp[3].end());
    if(ans > int(4e8))
        ans = -1;
    cout << ans << endl;
}

1487F - Ones

Idea: Neon

Tutorial

1487F - Ones

Let's build the number from the lowest digit to the highest digit with the following dynamic programming: dpi,carry,cp,cndpi,carry,cp,cn — the minimum number of ones, if ii least significant digits are already fixed, the carry to the next digit is carrycarry (can be negative), there are cpcp positive numbers (of the form 111⋯111111⋯111) of length greater than or equal to ii and cncn negative numbers of length greater than or equal to ii.

First, consider the transitions when we reduce the values of cpcp and/or cncn. Such transitions correspond to the fact that in the optimal answer there were several numbers of length exactly ii, and they should not be considered further.

If the value of (cp−cn+carry)mod10(cp−cn+carry)mod10 matches the ii-th least significant digit in nn, then we can use transition to (i+1)(i+1)-th state with the new value of carrycarry and the number of ones in the answer increased by cp+cncp+cn.

It remains to estimate what the maximum value of cpcp (cncn) and carrycarry we need. The value of cpcp doesn't exceed the total number of numbers that we use in the answer. Using at most 55 numbers we can decrease the length of nn by at least 11. Thus, the maximum value of cpcp and cncn is at most 5|n|5|n| (where |n| is the length of the number nn). For the value of carrycarry, the condition carry≥carry+cp10carry≥carry+cp10 should be met (similarly for a negative value). Thus, we can assume that the absolute value of carrycarry doesn't exceed 5|n|95|n|9.

The total complexity of this solution is O(|n|4)O(|n|4), yet with a high constant factor.

Solution (Neon)
#include <bits/stdc++.h>

using namespace std;

#define sz(a) int((a).size())
#define forn(i, n) for (int i = 0; i < int(n); ++i)

const int N = 250;
const int M = 28;
const int INF = 1e9;

int dp[2][M * 2 + 1][N][N];

int main() {
  string s;
  cin >> s;
  reverse(s.begin(), s.end());
  s += "0";
  
  forn(carry, M * 2 + 1) forn(cp, N) forn(cn, N) dp[0][carry][cp][cn] = INF;
  dp[0][M][N - 1][N - 1] = 0;
  
  forn(i, sz(s)) {
    forn(carry, M * 2 + 1) forn(cp, N) forn(cn, N) dp[1][carry][cp][cn] = INF;
    forn(carry, M * 2 + 1) for (int cp = N - 1; cp >= 0; --cp) for (int cn = N - 1; cn >= 0; --cn) if (dp[0][carry][cp][cn] != INF) {
      if (cp > 0) dp[0][carry][cp - 1][cn] = min(dp[0][carry][cp - 1][cn], dp[0][carry][cp][cn]);
      if (cn > 0) dp[0][carry][cp][cn - 1] = min(dp[0][carry][cp][cn - 1], dp[0][carry][cp][cn]);
      int rcarry = carry - M;
      int val = rcarry + cp - cn;
      int digit = val % 10;
      if (digit < 0) digit += 10;
      int ncarry = val / 10;
      if (val < 0 && digit != 0) --ncarry;
      if (digit == s[i] - '0') dp[1][ncarry + M][cp][cn] = min(dp[1][ncarry + M][cp][cn], dp[0][carry][cp][cn] + cp + cn);
    }
    swap(dp[0], dp[1]);
  }
  
  int ans = INF;
  forn(i, N) forn(j, N) ans = min(ans, dp[0][M][i][j]);
  cout << ans << endl;
}

1487G - String Counting

Idea: BledDest

Tutorial

1487G - String Counting

Suppose there is no constraint on the number of letters used. Then this problem can be solved with the following dynamic programming: let dpi,j,kdpi,j,k be the number of strings of length ii ending with characters jj and kk that don't contain palindromes of odd length greater than 11 (obviously, each forbidden palindrome contains a subpalindrome of length 33, so we only need to ensure that there are no palindromes of length 33).

The thing we are going to use in order to ensure that all the constraints on the number of characters are met is inclusion-exclusion. Since each ci>n3ci>n3, at most two characters can violate their constraints in a single string, so we will iterate on some character of the alphabet and subtract the number of strings violating the constraint on this character from the answer, then iterate on a pair of characters and add the number of strings violating the constraints on these two characters to the answer.

Okay, how to calculate the number of strings violating the constraint on some fixed character? Let's use dynamic programming fi,j,k,lfi,j,k,l — the number of strings such that they contain ii characters, jj of them have the same type that we fixed, the previous-to-last character is kk and the last character is ll. The number of states here seems to be something about n2⋅262n2⋅262, but in fact, kk and ll can be optimized to have only two different values since we are interested in two types of characters: the ones that coincide with the character we fixed, and the ones that don't.

Okay, what about violating the constraints on two characters? The same method can be used here: let gi,j,k,l,mgi,j,k,l,m be the number of strings consisting of ii characters such that the number of occurrences of the first fixed character is jj, the number of occurrences of the second fixed character is kk, the previous-to-last character is ll and the last character is mm. Again, at first it seems that there are up to n3⋅262n3⋅262 states, but ll and mm can be optimized to have only 33 different values, so the number of states is actually n3⋅32n3⋅32.

It seems that we have to run this dynamic programming for each pair of characters, right? In fact, no, it is the same for every pair of characters, the only difference is which states violate the constraints and which don't. We can run this dp only once, and when we need an answer for the pair of characters (x,y)(x,y), we can use two-dimensional prefix sums to query the sum over gi,j,k,l,mgi,j,k,l,m with i=ni=n, j>cxj>cx and k>cyk>cy in O(1)O(1). In fact, this dynamic programming can also be used for the first and the second part of the solution (calculating the strings that don't violate any constraints and the strings that violate the constraints on one character), so the hardest part of the solution runs in O(n3)O(n3), though with a pretty big constant factor.

Solution (BledDest)
#include<bits/stdc++.h>

using namespace std;

const int MOD = 998244353;
const int N = 402;

int add(int x, int y)
{
    x += y;
    while(x >= MOD) x -= MOD;
    while(x < 0) x += MOD;
    return x;
}

int sub(int x, int y)
{
    return add(x, -y);
}

int mul(int x, int y)
{
    return (x * 1ll * y) % MOD;
}

int c[26];
int dp[2][N][N][3][3];
int sumDp[N][N];
int p1[N][N];
int p2[N][N];
int n;

int main()
{
    cin >> n;
    for(int i = 0; i < 26; i++)
        cin >> c[i];
    for(int i = 0; i < 26; i++)
        for(int j = 0; j < 26; j++)
            for(int k = 0; k < 26; k++)
                if(i != k)
                {                                                                    
                    multiset<int> s = {i, j, k};
                    dp[1][s.count(0)][s.count(1)][min(2, j)][min(2, k)]++;
                }
    for(int i = 4; i <= n; i++)
    {
        for(int j = 0; j < N; j++)
            for(int k = 0; k < N; k++)
                for(int x = 0; x < 3; x++)
                    for(int y = 0; y < 3; y++)
                    {
                        dp[0][j][k][x][y] = dp[1][j][k][x][y];
                        dp[1][j][k][x][y] = 0;
                    }
        for(int j = 0; j < N; j++)
            for(int k = 0; k < N; k++)
                for(int x = 0; x < 3; x++)
                    for(int y = 0; y < 3; y++)
                    {
                        int cur = dp[0][j][k][x][y];
                        if(cur == 0) continue;
                        for(int z = 0; z < 3; z++)
                        {                      
                            int& nw = dp[1][j + (z == 0 ? 1 : 0)][k + (z == 1 ? 1 : 0)][y][z];
                            nw = add(nw, mul(cur, (z == 2 ? 24 : 1) - (z == x ? 1 : 0)));
                        }
                    }
    }
    for(int i = 0; i < N; i++)
        for(int j = 0; j < N; j++)
            for(int k = 0; k < 3; k++)
                for(int l = 0; l < 3; l++)
                    sumDp[i][j] = add(sumDp[i][j], dp[1][i][j][k][l]);
    for(int i = 0; i < N; i++)
    {
        p1[i][N - 1] = sumDp[i][N - 1];
        for(int j = N - 2; j >= 0; j--)
            p1[i][j] = add(sumDp[i][j], p1[i][j + 1]);
    }
    for(int j = 0; j < N; j++)
    {
        p2[N - 1][j] = p1[N - 1][j];
        for(int i = N - 2; i >= 0; i--)
            p2[i][j] = add(p1[i][j], p2[i + 1][j]);
    }
    int ans = p2[0][0]; 
    for(int i = 0; i < 26; i++)
    {
        for(int j = 0; j < N; j++)
            for(int k = c[i] + 1; k < N; k++)
                ans = sub(ans, sumDp[k][j]);
    }
    for(int i = 0; i < 26; i++)
        for(int j = 0; j < i; j++)
            ans = add(ans, p2[c[i] + 1][c[j] + 1]);
    cout << ans << endl;    
}

In problem E, I have a bit of a different solution to the DP optimization. From the previous course, we sort all dp values from low to high. Then for a certain new dish, we want to know the first option in the sorted list that isn't forbidden. First we assume every dish will pick the first option, and we put the option they choose in an array (initially all zeroes). Then go through all bad edges of the first dp option and increment the lowest possible dp option for each bad dish, then do the same for the bad edges of the second option, etc. Then it is simple to calculate the new dp values. This algorithm takes only O(n⋅log(n)+m)O(n⋅log(n)+m), because it goes through each edge once, and does a sort on the dp values.

    Of course, 3 * n mod n = 0 and that's why 3 * n * (n — 1) mod n = 0. But after dividing 3 * n * (n — 1) by 2, this condition is not necessarily satisfied (for example, with n = 2: 3 * 2 * 1 = 6, 6 mod 2 = 0. But (3 * 2 * 1) / 2 = 3, 3 mod 2 != 0). Also statement 3 * n * (n — 1) / 2 mod n = n / 2 is correct if and only if n is even (if it is odd left part is integer and right is not). proof: 3 * n * (n — 1) / 2 mod n = n / 2 <=> 3 * n * (n — 1) / 2 — n / 2 mod n = 0 <=> (3 * n * n — 4 * n) / 2 mod n = 0 <=> n * (3 * n — 4) / 2 mod n = 0 — this is correct because 3 * n — 4 is even (because n is even) and (3 * n — 4) / 2 is integer.

        Yes. You can do it in following way: We want to find a number from 0 to n — 1 inclusive such that 3 * n * (n — 1) / 2 mod n = x. 3 * n * (n — 1) / 2 mod n = x <=> 3 * n * (n — 1) / 2 — x mod n = 0 <=> 3 * n * n — 3 * n — 2 * x mod 2 * n = 0. It means (1) that 3 * n + 2 * x mod 2 * n = 0 (because 3 * n * n mod 2 * n = 0, as n is even and 0 mod 2 * n = 0) and (2) 2 * x mod n = 0 (because 3 * n * n — 3 * n mod n = 0 and 0 mod n = 0). It follows from (2) that x = 0 or x = n / 2 (because 0 <= x < n). If x = 0, (1) sometimes is false (for example, if n = 2). If x = n / 2, 3 * n + 2 * x = 4 * n, 4 * n mod 2 * n = 0. So the only solution to this problem is x = n / 2.
